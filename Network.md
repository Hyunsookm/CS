# Network

## 네트워크 개요
#### 네트워크의 정의와 필요성
- 정의:
  - 통신 및 데이터 교환을 용이하게 하기 위해 함께 연결된 컴퓨터, 서버, 라우터, 스위치 및 기타 네트워킹 장비와 같은 상호 연결된 장치의 모음이다.
- 사용 목적:
  - 프로그램이나 데이터를 공유하여 사용할 수 있다.
  - 프린터 등 주변 장치를 공유하여 사용할 수 있다.
  - 인터넷 회선을 공유하여 여러 대의 컴퓨터가 편리하게 인터넷을 사용할 수 있다.
- 장점:
  - 네트워크를 통해 방대한 자료를 공유할 수 있다.
  - 언제 어디서든 리소스에 접근할 수 있다.
  - IoT와 같은 최신 기술이 네트워크를 통해 연결되어, 다양한 응용 프로그램과의 통합을 가능하게 한다.
#### 데이터 통신의 기본 구성 요소
##### 송신자(Sender), 수신자(Receiver), 메시지(Message), 전송 매체(Medium), 프로토콜(Protocol)
1. 송신자(Sender)
   - 정의:
     - 데이터 메시지를 전송하는 장치
   - 예시:
     - 컴퓨터, 스마트폰, 서버

2. 수신자(Receiver)
   - 정의:
     - 메시지를 받는 장치
   - 예시:
     - 컴퓨터, 스마트폰, 서버

3. 메시지(Message)
   - 정의:
     - 송신자가 보내는 실제 데이터
   - 예시:
     - 텍스트, 이미지, 비디오

4. 전송 매체(Medium)
   - 정의:
     - 송신자에서 수신자로 이동하는 물리적인 경로
   - 예시:
     - 꼬임쌍선, 동축선, 광케이블

5. 프로토콜(Protocol)
   - 정의:
     - 상호 합의 된, 데이터 통신을 통제하는 규칙들의 집합
   - 예시:
     - TCP, UDP, HTTP
#### 네트워크 모델 (Network Models)	
##### OSI 7계층 모델 - Physical, Data Link, Network, Transport, Session, Presentation, Application
- 정의:
  - 국제표준화기구(ISO)에서 개발한 모델로, 컴퓨터 네트워크 프로토콜 디자인과 통신을 계층으로 7단계로 나누어 설명한 것이다.
![image](/image/osi7layers.png)
1. Physical Layer(물리 계층)
- 정의:
  - 장치 간 전기적 신호를 전달하는 계층이며, 데이터 프레임 내부의 각 bit를 한 노드에서 다음 노드로 실제로 이동시키는 계층이다.
- 특징:
  - 통신 데이터의 단위는 bit
  - 대표적인 장비는 통신 케이블, 허브, 리피터가 있다.
  - 데이터를 단지 전기적인 신호로 변환하여 주고받는 기능만 수행함.

2. Datalink Layer(물리 계층)
- 정의:
  - 물리계층을 통해 송수신되는 정보의 오류와 흐름을 관리하여 안전한 정보의 전달을 수행할 수 있도록 도와주는 역할을 하는 계층이다.
- 특징:
  - 통신 데이터의 단위는 frame
  - hop-to-hop 전송을 함.
  - MAC 주소를 가지고 통신을 한다.
  - 대표적인 장비는 스위치, 브릿지가 있다.
  - 대표적인 프로토콜은 HDLC, ADCCP, LLC, ALOHA

3. Network Layer(물리 계층)
- 정의:
  - 경로를 선택하고 주소를 정하고 경로에 따라 패킷을 전달해주는 역할을 하는 계층이다.
- 특징:
  - 통신 데이터의 단위는 packet
  - 라우팅, 흐름 제어, 세그멘테이션 등의 기능을 수행한다.
  - 대표적인 장비로는 라우터, 스위치가 있다.
  - 대표적인 프로토콜은 IP, ICMP, RIP

4. Transport Layer(물리 계층)
- 정의:
  - 데이터 전송의 오류 검출, 수정 및 흐름 제어를 수행하는 계층이다.
- 특징:
  - 통신 데이터의 단위는 segment
  - 데이터 전송을 위해 Port 번호를 사용한다.
  - 송신자와 수신자 간의 신뢰성있고 효율적인 데이터를 전송하기 위하여 오류검출 및 복구, 흐름제어와 중복검사 등을 수행
  - 대표적인 프로토콜은 TCP, UDP, SCTP

5. Session Layer(물리 계층)
- 정의:
  - 응용 프로그램 간의 대화를 유지하기 위한 구조를 제공하고, 이를 처리하기 위해 프로세스들의 논리적인 연결을 담당하는 계층이다.
- 특징:
  - 통신 중 연결이 끊어지지 않도록 유지시켜주는 역할 수행하기 위해 TCP/IP 세션 연결의 설정과 해제, 세션 메세지 전송 등의 기능을 수행한다.
  - 통신을 하기 위한 세션 확립, 유지, 중단을 수행한다.
  - 통신하는 사용자들을 동기화해주고, 오류 복구 명령들을 일괄적으로 처리한다.
  - 대표적인 프로토콜은 PRC, NetBIOS

6. Presentation Layer(물리 계층)
- 정의:
  - 애플리케이션이 다루는 정보를 통신에 알맞은 형태로 만들거나, 하위 계층에서 온 데이터를 사용자가 이해할 수 있는 형태로 만드는 역할을 담당하는 계층이다.
- 특징:
  - 데이터의 효율과 보안을 위해 압축과 암호화를 수행하고, 전송을 위한 포맷으로 변경을 수행한다.
  - 대표적인 프로토콜은 JPEG, MPEG이다.

7. Application Layer(물리 계층)
- 정의:
  - 응용 프로세스와 직접 관계하여 일반적인 응용 서비스를 수행하는 역할을 담당하는 계층이다.
- 특징:
  - 응용 프로세스 간의 정보 교환, 파일 전송 등의 서비스를 제공한다.
  - 대표적인 장비는 L7 스위치
  - 대표적인 프로토콜은 HTTP, FTP, SMTP, POP3


##### TCP/IP 모델
- TCP/IP 모델은 IETF라는 단체에서 제정하였다.
![image](/image/tcpip.png)
1. Network Interface(네트워크 인터페이스 계층)
- 정의:
  - 물리적인 네트워크 매체를 통해 데이터를 전송하는 기능을 담당하는 계층이다.
- 특징:
  - 이더넷, Wi-Fi 등 다양한 LAN 및 WAN 기술을 지원한다.
  - 하드웨어 주소(MAC 주소)를 사용하여 네트워크 장치를 식별한다.

2. Internet(인터넷 계층)
- 정의:
  - 패킷의 라우팅 및 주소 지정을 담당하고, 서로 다른 네트워크 간의 데이터 전송을 가능하게 하는 계층이다.
- 특징:
  - IP, ICMP, ARP 프로토콜을 지원한다.

3. Transport(전송 계층)
- 정의:
  - 애플리케이션 간의 통신을 관리하고, 흐름 제어 및 오류 검출 기능을 하는 계층이다.
- 특징:
  - TCP, UDP 프로토콜을 지원한다.

4. Application(응용 계층)
- 정의:
  - 사용자 요청을 처리하고, 데이터 전송을 위해 필요한 모든 기능을 수행하는 계층이다.
- 특징:
  - HTTP, FTP, SMTP 프로토콜을 지원한다.
  - 사용자 인터페이스와 가장 가까운 계층으로, 데이터의 형식을 정의한다.
  - 클라이언트-서버 모델을 기반으로 데이터 송수신을 처리한다.

#### 네트워크의 종류 (Types of Networks)

##### LAN(Local Area Network), WAN(Wide Area Network), MAN(Metropolitan Area Network), PAN(Personal Area Network)
![image](/image/network.png)
1. LAN(Local Area Network)
- 정의:
  - 근거리 통신망으로, 지역적인 범위에서 컴퓨터와 기타 장치들이 서로 연결되어 데이터를 공유하는 네트워크이다.
- 특징:
  - 조직내부나 동일 건물등 비교적 좁은 지역을 연결하기 위해 사용한다.
  - 초기 투자비용이 많이들고 유지비용은 적게든다.
  - 관리자가 직접 관리하는 방식

2. WAN(Wide Area Network)
- 정의:
  - 넓은 지리적 범위(도시, 국가, 대륙 등)에 걸쳐 여러 LAN을 연결하는 네트워크이다.
- 특징:
  - 일반적으로 공공 통신 인프라를 통해 데이터 전송을 수행한다.
  - 수백 킬로미터에서 수천 킬로미터까지의 범위를 커버한다.
  - 위성, 전화선, 광섬유 등 다양한 전송 매체를 사용하여 연결된다.
  - 설치 및 운영 비용이 상대적으로 높다.
- 예시:
  - 기업의 자사 간 연결
  - 국가 및 국제 데이터 센터 간 연결

3. MAN(Metropolitan Area Network)
- 정의:
  - 특정 도시나 대도시 지역 내에서 여러 LAN을 연결하는 네트워크이다.
- 특징:
  - 일반적으로 수십킬로미터 범위 내에서 운영된다.
  - WAN보다 좁고, LAN보다 넓은 범위를 커버한다.
  - WAN보다 빠른 데이터 전송 속도를 제공한다.
- 예시:
  - 도시 내의 기업 및 기관 간 연결
  - 대학 캠퍼스 네트워크
  - 공공 서비스 및 정부 기관의 네트워크

4. PAN(Personal Area Network)
- 정의:
  - 개인의 장치 간에 데이터를 전송하기 위해 사용되는 소규모 네트워크이다.
- 특징:
  - 일반적으로 개인이 소지한 장치 간의 연결을 의미한다.
  - 10~30m 정도의 짧은 범위를 커버한다.
  - 주로 Bluetooth, Wi-Fi Direct, Zigbee 등이 사용된다.
- 예시:
  - 개인 기기 간 데이터 공유
  - IoT 장치 간의 통신
  - 블루투스 스피커와 스마트폰의 연결

##### WLAN(Wireless LAN), SAN(Storage Area Network)
1. WLAN(Wireless LAN)
- 정의:
  - 유선 연결 대신 무선 기술을 사용하여 로컬 네트워크 노드를 연결하는 근거리 통신망(LAN)의 한 유형이다.
- 특징:
  - 무선 주파수(RF) 또는 적외선 기술을 사용하여 데이터를 공중으로 전송하고 수신한다.
  - 케이블 없이 디바이스를 네트워크에 연결한다.
  - 설치, 유지보수, 재배치가 간편하다.
  - 액세스 포인트(AP)를 이용하여 신호를 증폭하고 인터넷에 연결한다.
- 예시:
  - 가정용 Wi-Fi 네트워크
  - 사무실 무선 네트워크
  - 공공 장소의 Wi-Fi 핫스팟 (카페, 호텔, 공항 등)

2.SAN(Storage Area Network)
- 정의:
  - 여러 컴퓨터와 서버가 공유할 수 있는 고속 전용 스토리지 네트워크이다.
- 특징:
  - 일반 네트워크와 분리된 전용 스토리지 네트워크로 동작한다.
  - 블록 단위 데이터 전송을 지원하여 빠르고 안정적인 데이터 접근을 제공한다.
  - 스토리지 디바이스들이 네트워크로 연결되어 중앙화된 논리 볼륨을 형성한다.
- 예시:
  - 데이터베이스 서버
  - 가상화 배포
  - 대규모 가상 데스크톱 인프라

#### 네트워크 토폴로지 (Network Topology)
- 정의:
  - 네트워크를 배치하는 방식으로, 링크와 노드 들이 어떤 식으로 배치되어 서로 연결되는지를 물리적으로 혹은 논리적으로 설명하는 것이다.
##### 스타(Star), 버스(Bus), 링(Ring), 메시(Mesh), 트리(Tree)
1. 스타(Star)
- 정의:
  - 가장 흔하게 쓰이는 네트워크 토폴로지로, 네트워크 안의 모든 노드가 동축 케이블이나 연선 또는 광케이블을 통해 직접 하나의 중앙 허브에 연결된 형식이다.
- 특징:
![image](/image/star-topology.png)
  - 중앙 노드는 서버로서 작동하여 데이터의 흐름을 관리합니다.
  - 각각의 노드에서 보낸 정보가 목적지에 닿기 위해서는 반드시 중앙 노드를 지나야 한다.
  - 중앙 노드는 리피터처럼 데이터 손실을 막아준다.
- 장점:
  - 한 장소에서 전체 네트워크를 관리할 수 있다.
  - 각 노드는 독립적으로 연결되어 있어, 노드 하나의 장애가 발생해도 다른 노드는 운용 가능하다.
  - 장애가 발생한 경우 원인을 쉽게 찾을 수 있다.
- 단점:
  - 중앙 허브에 문제가 생기면 전체 네트워크가 모두 멈춘다.
  - 전체 네트워크의 성능이 중앙 노드에 의해 결정된다.

- 예시:
  - 가정용 네트워크
  - 소규모 사무실 네트워크

2. 버스(Bus)
- 정의:
  - 네트워크상의 모든 장치가 하나의 케이블로 연결되어 있는 유형이다.
- 특징:
![image](/image/bus-topology.png)
  - 네트워크의 한쪽 끝에서 다른 쪽 끝으로 한 방향으로 연결되어 있기에 “선형 토폴로지" 또는 “백본 토폴로지"라고도 지칭한다.
  - 데이터의 흐름이 케이블의 경로를 따라 한 방향으로 이동한다.
- 장점:
  - 비용이 효율적이고, 소규모 네트워크에 적합하다.
  - 장치의 추가, 제거가 쉽다.
- 단점:
  - 케이블에 장애가 발생할 경우, 모든 네트워크가 중단된다.
  - 노드를 추가할 때마다, 대역폭이 낭비된다.
  - 양 방향 통신이 불가능하다.
- 예시:
  - 이벤트, 전시회 등의 임시 네트워크


3. 링(Ring)
- 정의:
  - 노드들이 원 또는 고리 모양으로 배열되어 있는 유형이다.
- 특징: 

    ![image](/image/ring-topology.png)
  - 데이터가 원 모양의 네트워크를 따라 한 방향 또는 양방향으로 흐른다.
  - 각 노드의 양 옆에는 두 개의 이웃 노드가 존재한다.
- 장점:
  - 한 번에 하나의 노드에서만 데이터를 전송할 수 있어 패킷 충돌의 위험이 낮다.
  - 설치 비용이 낮고, 문제 발생 시 원인 파악이 쉽다.
- 단점:
  - 한 노드에 문제 발생 시 모든 네트워크가 중단된다.
  - 노드 추가, 제거 시에 전체 네트워크가 중단된다.
- 예시:
  - LAN에서 주로 사용한다.

4. 매시(Mesh)
- 정의:
  -  노드들을 점 대 점으로 상호 연결한 구조로 복잡하고 정교한 유형이다.
- 특징:

    ![image](/image/mesh-topology.png)
  - 모든 노드가 상호연결되어 있다.
  - 라우팅 방식으로 전송 시, 최단 거리를 결정하여 데이터를 전송한다.
  - 플러딩 방식으로 전송 시, 네트워크 내부의 모든 노드에 데이터를 전송한다.
- 장점:
  - 안정성과 보안성이 높다.
  - 노드 간에 상호 연결된 정도가 높고 복합적이어서 하나의 노드에 문제가 발생해도, 나머지는 운용 가능하다.
- 단점:
  - 네트워크 관리, 설치, 재구성이 어렵다.
  - 케이블 등 설치 비용이 많이 든다.
- 예시:
  - WAN에서 주로 사용되고, 군사용 네트워크 등 안정한 통신이 필요한 분야에서 사용.
  - IoT, 블록체인 네트워크 구성

5. 트리(Tree)
- 정의:
  - 노드들이 부모-자식 계층구조로 연결된 유형이다.
- 특징:

    ![image](/image/treetopology.png)
  - 모든 노드가 상호연결되어 있다.

- 장점:
  - 네트워크의 확장이 매우 쉽다.
  - 성능에 문제가 생긴 경우, 각 노드에 개별적으로 접근할 수 있어 문제 해결이 쉽다.
- 단점:
  - 상위 계층의 장치에 문제 발생 시, 하위 계층의 장치도 영향을 받을 수 있다.
  - 구조가 복잡하다.
- 예시:
  - 대규모 기업 네트워크
  - 캠퍼스 네트워크

## 데이터 통신 및 전송 기술

#### 데이터 표현과 신호 (Data Representation and Signals)

##### 디지털 데이터 - 이진 신호, 샘플링
1. 이진 신호(Binary Signal)
- 정의:
  - 0과 1로 이루어진 비연속적이고, 이산 시간 신호이다.
- 특징:
  - 이진 형식으로 데이터를 전송한다.
  - 노이즈에 대한 저항력이 강해 오류 발생 가능성이 낮다.
  - 컴퓨터와 디지털 장치에서 쉽게 처리할 수 있다.

2. 샘플링
- 정의:
  - 아날로그 신호의 연속적인 값을 일정한 시간 간격으로 추출하여 이산적인 값으로 변환하는 과정이다.
- 특징:
  - 원래의 데이터에서 샘플을 추출하여, 추출한 데이터만 사용하는 방법이다.
##### 아날로그 신호 - 주파수, 진폭, 위상
1. 주파수
- 정의:
  - 신호가 단위 시간동안 반복되는 횟수를 나타내는 물리량이다.
- 특징:
  - 일반적으로 단위는 Hz를 사용한다.
  - 주파수가 높을 수록 주기적인 현상이 빈번하게 발생한다.

2. 진폭
- 정의:
  - 신호의 최대 변동 범위를 나타낸다.
- 특징:
  - 신호의 강도나 세기를 의미한다.
  - 아날로그 신호에서는 주로 전압의 크기로 표현한다.
  - 진폭이 클수록 신호의 강도가 강하다.

3. 위상
- 정의:
  - 주기적인 신호의 특정 시점에서의 위치를 나타내는 각도이다.
- 특징:
  - 주로 라디안이나 도(degree) 단위로 표현된다.
  - 서로 다른 위상을 가진 신호가 겹칠 경우, 간섭 현상이 발생할 수 있다.

#### 부호화와 변조 (Encoding and Modulation)
1. 부호화(Encoding)
- 정의:
  - 데이터를 다른 형식으로 변환하는 과정이다.
- 특징:
  - 입력한 문자나 기호를 컴퓨터가 읽고 처리할 수 있는 형태로 변환한다.
  - 데이터를 효율적으로 전송하거나 저장하기 위한 목적으로 사용한다.
- 예시:
  - 문자 인코딩(UTF-8, ASCII)
  - URL 인코딩(URL에서 사용될 수 없는 특수 문자는 %에 숫자를 붙여 인코딩한다.)

2. 변조(Modulation)
- 정의:
  - 인코딩(부호화)된 데이터를 원래의 형식으로 변환하는 과정이다.
- 특징:
  - 데이터를 복원하여 사람이 이해하고 처리할 수 있는 형태로 변환하는 작업을 한다.
  - 이진수 데이터를 사람이 처리할 수 있는 문자로 변환한다.
- 예시:
  - ASCII 디코딩(65 -> 'A')
  - Base64 디코딩(SGVsbG8sIFdvcmxkIQ== -> "Hello, World")

##### 라인 부호화(Line Coding), 주파수 변조(FM), 진폭 변조(AM), 위상 변조(PM)
1. 라인 부호화(Line Coding)
- 정의:
  - 디지털 데이터를 디지털 신호로 바꾸는 것이다.
- 특징:
  - 이진 데이터를 전기 신호로 변환하여 전송한다.
  - 연속적인 신호에서 비트를 구분할 수 있도록 설계해야, 수신측에서 정확하게 해석할 수 있다.
  - 대칭적인 신호 형태를 제공하여, 신호의 전송 과정에서 발생할 수 있는 왜곡이나 변형을 최소화한다.
- 예시:
  - NRZ, RZ, Manchester Encoding

2. FM(Frequency Modulation, 주파수 변조)
- 정의:
  - 정보 신호의 주파수를 변조하여 전송하는 방식이다.
- 특징:
  - 잡음에 대한 저항력이 강해 신호 품질이 우수하다.
  - FM은 넓은 대역폭을 필요로 하며, 나이퀴스트 정리에 따라 최대 주파수의 두 배 이상이 필요하다.
- 예시:
  - 라디오, TV, 무선 통신


3. AM(Amplitude Modulation, 진폭 변조)
- 정의:
  - 정보 신호의 진폭을 변조하여 전송하는 방식이다.
- 특징:
  - 아날로그 신호의 진폭이 변할 때, 그에 따라 반송파의 진폭도 변한다.
  - AM 신호는 간섭과 잡음에 취약하여 신호 품질이 저하될 수 있다.
  - AM은 대역폭이 넓어, 두 배의 변조 주파수 대역을 필요로 한다.
- 예시:
  - 라디오

4. PM (Phase Modulation, 위상 변조)
- 정의:
  - 정보 신호의 위상을 변조하여 전송하는 방식이다.
- 특징:
  - PM은 FM과 유사하게 잡음에 대한 저항력이 강하다.
  - 주파수 변조와 마찬가지로 나이퀴스트 정리에 따른 고려가 필요하다.
- 예시:
  - QPSK(Quadrature Phase Shift Keying)

##### 다중화 기술 - TDM(Time Division Multiplexing), FDM(Frequency Division Multiplexing), CDM(Code Division Multiplexing)
1. TDM (Time Division Multiplexing, 시간 분할 다중화)
- 정의:
  - 한 전송로에 주어진 시간대역을 여러 타임슬롯으로 분할하여 각 채널에 배당함으로써 많은 채널이 하나의 전송회선을 공유하는 방식이다.
- 특징:
  ![image](/image/tdm.png)
  - 채널간 간섭을 막기 위해 보호시간 필요
  - 여러 다른 채널로부터 정보를 샘플링하여 채널 사이에 시차를 두어 하나의 회선으로 다수의 채널을 전송한다.


2. FDM (Frequency Division Multiplexing, 주파수 분할 다중화)
- 정의:
  - 전송되어야 하는 신호들의 대역폭을 합한 것보다 링크의 대역폭이 클 때 적용할 수 있는 아날로그 기술로, 한 전송로의 대역폭을 여러 개의 작은 채널로 분할하여 여러 단말기가 동시에 이용하는 방식이다.
- 특징:
  ![image](/image/fdm.png)
  - 전송되어야 하는 신호들의 대역폭을 합한 것보다 링크의 대역폭이 클 때 적용할 수 있는 아날로그 기술이다.
  -  각 장치로부터 받은 신호를 각기 다른 반송 주파수로 변조하여, 링크를 통해 이동할 수 있는 하나의 복합신호로 합쳐지게 된다.


3. CDM (Code Division Multiplexing, 코드 분할 다중화)
- 정의:
  - 여러 신호를 서로 다른 코드로 인코딩하여 동시에 전송하는 방식이다
- 특징:
  - 모든 신호가 동일한 주파수 대역에서 동시에 전송되지만, 각 신호는 고유한 코드로 구분된다.
  - 코드의 특성으로 인해 잡음에 대한 저항력이 강하다.
  - FDM과 TDM의 혼합 방식이다.
- 예시:
  - CDMA와 같은 무선 통신 시스템에서 널리 사용된다.


#### 전송 방식 (Transmission Methods)	

##### 직렬 전송(Serial Transmission), 병렬 전송(Parallel Transmission)
1. 직렬 전송(Serial Transmission)
- 정의:
  - 데이터를 한 비트씩 순차적으로 전송하는 방식이다.
- 특징:
  - 통신 회선을 통하여 한 번에 한 비트씩 순서대로 전송하는 방식
  - 전송속도는 느리지만, 통신 회선의 비용이 저렴하다.
- 예시:
  - 이더넷, HDMI 등의 원거리 통신

2. 병렬 전송(Parallel Transmission)
- 정의:
  - 별렬로 나열하여, 한 번에 n비트 이상의 데이터를 전송하는 방식이다.
- 특징:
  - 여러 개의 전송로를 통하여 동시에 여러 비트를 전송한다.
  - 직렬 전송에 비해 속도는  빠르나, 통신회선을 구축하는 데 많은 비용이 소요된다.
- 예시:
  - IC회로, RAM 등 근거리 통신에서 사용한다.
##### 동기식 전송(Synchronous)과 비동기식 전송(Asynchronous)

## 네트워크 프로토콜

#### 프로토콜의 개념과 역할 (Concept and Role of Protocols)	
- 정의:
  - 컴퓨터 네트워크에서 장치 간 데이터를 주고 받는 데 사용되는 합의된 규칙/규약
- 역할:
  - 데이터가 어떻게 구조화되고 인코딩되는지를 정의하여, 수신자가 데이터를 올바르게 해석할 수 있도록 한다.
  - 데이터가 전송되는 방식(예: 신뢰성 있는 전송, 비신뢰성 전송 등)을 규정하여, 효율적인 데이터 흐름을 보장한다.
  - 전송 중 발생할 수 있는 오류를 감지하고 수정하는 방법을 정의하여, 데이터의 무결성을 유지한다.
#### 데이터 링크 계층 프로토콜 (Data Link Layer Protocols)

##### 이더넷(Ethernet), PPP(Point-to-Point Protocol), HDLC(High-Level Data Link Control)
1. 이더넷(Ethernet)
- 정의:
  - 컴퓨터 네트워크 간의 데이터 통신 기술로, 주로 LAN에서 사용되는 프로토콜이다.
- 특징:
  - 허브, 스위치 등 같은 장비에 연결된 같은 네트워크 내부에서의 데이터 전송에서 사용한다.
  - 데이터 링크 계층에서 MAC 주소와 프로토콜의 형식을 정의한다.
  - 광섬유 케이블, 구리선 등의 물리적 매체를 통해 데이터를 전송할 수 있다.
  - CSMA/CD 방식을 사용하여, 네트워크에서 여러 장비가 동시에 데이터를 전송할 때 발생할 수 있는 충돌을 감지한다.
- frame 구조:
  ![image](/image/ethernetframe.png)
  https://velog.io/@moonblue/%EC%9D%B4%EB%8D%94%EB%84%B7-%ED%94%84%EB%A0%88%EC%9E%84-Ethernet-Frame
  - Preamble: 프레임의 시작을 알리는 동기화 비트 패턴으로, 수신 측에서 프레임의 시작을 인식할 수 있게한다.
  - Destination address: 수신할 장치의 MAC 주소를 나타낸다.
  - Source address: 데이터를 전송한 장치의 MAC 주소를 나타낸다.
  - Type: 프레임 내의 페이로드 데이터 형식을 식별하는 필드로, 상위 계층 프로토콜(예: IPv4, IPv6)의 종류를 알려준다.
  - FCS(Frame Check Sequence): 프레임의 무결성을 확인하기 위한 오류 검출 코드로, 수신 측에서 프레임의 오류를 검출한다.

2. PPP(Point-to-Point Protocol)
- 정의:
  - 두 지점 간의 데이터 통신을 위한 프로토콜이다.
- 특징:
  - HDLC 프로토콜의 프레임에 프로토콜 필드를 추가한 프레임 구조를 가진다.
  - IP 등 여러 상위 프로토콜을 지원할 수 있게 설계됨.
  - FCS를 이용하여 데이터 전송 중 발생할 수 있는 오류를 검출한다.
- frame 구조:
  ![image](/image/ppp.png)
  - 앞, 뒤 필드에는 flag값으로 0x7e 값을 가지고, 처음과 끝을 의미한다.
  - 주소 필드에는 하나의 데이터링크에서 사용되므로 0xFF로 설정한다.
  - 제어 필드에는 0x03으로 비연결형 프레임을 나타낸다.
  - 프로토콜 필드에는 IP, LCP, NCP, PAP, CHAP를 구분하는 필드이다.
  - CRC 필드에는 checksum을 통해 오류를 감지하는 필드이다.

##### MAC(Media Access Control)과 프레임 처리
- 정의:
  - 네트워크 상의 장치들이 공유 매체에 접근할 때 충돌을 방지하고 효율적으로 통신할 수 있도록 하는 프로토콜이다.
- 특징:
  - 각 네트워크 장치는 고유한 MAC 주소를 가지고 있으며, 이를 통해 장치 간의 통신이 가능하다.
  - 여러 장치가 동시에 데이터를 전송할 경우 발생할 수 있는 충돌을 방지하는 규칙을 제공한다. 이더넷에서는 CSMA/CD 방식 사용한다.
  - MAC 계층은 데이터 프레임을 생성하고 전송하며, 수신된 프레임의 유효성을 검사한다.
  - 데이터가 전송될 때, 송신자의 프레임에는 송,수신자의 MAC 주소를 포함한다.
  - 데이터가 수신될 때, 수신자는 자신의 MAC 주소와 프레임의 MAC 주소를 비교하여 일치할 때만 데이터를 수신하고 추출하여 상위 계층으로 전달한다.

## 네트워크 설계와 관리 (Network Design and Management)

#### 네트워크 설계 원칙 (Principles of Network Design) - 확장성(Scalability), 신뢰성(Reliability), 비용 효율성(Cost Efficiency)
1. 확장성(Scalability)
- 정의:
  - 네트워크는 사용자 수나 트래픽 증가에 따라 쉽게 확장 가능해야 한다.
2. 신뢰성(Reliability)
- 정의:
  - 네트워크는 높은 가용성과 장애 발생 시 빠른 복구 능력을 가져야 한다.
3. 비용 효율성(Cost Efficiency)
- 정의:
  - 네트워크 설계는 산업 표준과 규정을 준수해야 한다.
#### IP 주소 체계 (IP Addressing System)
- 정의:
  - 인터넷에 연결되어 있는 각 장치를 식별하기 위해 TCP/IP 프로토콜 모음의 IP계층에서 사용하는 식별자이다.
- 특징:
  - IP 주소를 이용하여 인터넷 또는 로컬 네트워크를 통해 컴퓨터에 정보를 공유한다.
  - IPv4, IPv6 두 가지의 표준이 있다.
##### IPv4 서브넷팅 및 CIDR(Classless Inter-Domain Routing)
IPv4
- 정의:
  - 인터넷 프로토콜의 네 번째 버전이다.
- 특징:
  - 32비트 길이의 주소로, 총 2^32개의 주소를 가지고 있다.
  - 8비트씩 4자리로 표현하고, 각 자리는 0~255까지의 값을 가진다.

서브넷팅
- 정의:
  - 네트워크를 더 작은 단위의 네트워크로 분할하는 것을 말한다.
- 특징:
  - IP 주소의 일부를 네트워크 주소로 사용하고, 나머지를 호스트 주소로 사용하여 서브넷을 생성한다.
  - 서브넷 마스크를 사용하여 네트워크와 호스트를 구분한다.
