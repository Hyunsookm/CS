# Network

## 네트워크 개요
#### 네트워크의 정의와 필요성
- 정의:
  - 통신 및 데이터 교환을 용이하게 하기 위해 함께 연결된 컴퓨터, 서버, 라우터, 스위치 및 기타 네트워킹 장비와 같은 상호 연결된 장치의 모음이다.
- 사용 목적:
  - 프로그램이나 데이터를 공유하여 사용할 수 있다.
  - 프린터 등 주변 장치를 공유하여 사용할 수 있다.
  - 인터넷 회선을 공유하여 여러 대의 컴퓨터가 편리하게 인터넷을 사용할 수 있다.
- 장점:
  - 네트워크를 통해 방대한 자료를 공유할 수 있다.
  - 언제 어디서든 리소스에 접근할 수 있다.
  - IoT와 같은 최신 기술이 네트워크를 통해 연결되어, 다양한 응용 프로그램과의 통합을 가능하게 한다.
#### 데이터 통신의 기본 구성 요소
##### 송신자(Sender), 수신자(Receiver), 메시지(Message), 전송 매체(Medium), 프로토콜(Protocol)
1. 송신자(Sender)
   - 정의:
     - 데이터 메시지를 전송하는 장치
   - 예시:
     - 컴퓨터, 스마트폰, 서버

2. 수신자(Receiver)
   - 정의:
     - 메시지를 받는 장치
   - 예시:
     - 컴퓨터, 스마트폰, 서버

3. 메시지(Message)
   - 정의:
     - 송신자가 보내는 실제 데이터
   - 예시:
     - 텍스트, 이미지, 비디오

4. 전송 매체(Medium)
   - 정의:
     - 송신자에서 수신자로 이동하는 물리적인 경로
   - 예시:
     - 꼬임쌍선, 동축선, 광케이블

5. 프로토콜(Protocol)
   - 정의:
     - 상호 합의 된, 데이터 통신을 통제하는 규칙들의 집합
   - 예시:
     - TCP, UDP, HTTP
#### 네트워크 모델 (Network Models)	
##### OSI 7계층 모델 - Physical, Data Link, Network, Transport, Session, Presentation, Application
- 정의:
  - 국제표준화기구(ISO)에서 개발한 모델로, 컴퓨터 네트워크 프로토콜 디자인과 통신을 계층으로 7단계로 나누어 설명한 것이다.
![image](/image/osi7layers.png)
1. Physical Layer(물리 계층)
- 정의:
  - 장치 간 전기적 신호를 전달하는 계층이며, 데이터 프레임 내부의 각 bit를 한 노드에서 다음 노드로 실제로 이동시키는 계층이다.
- 특징:
  - 통신 데이터의 단위는 bit
  - 대표적인 장비는 통신 케이블, 허브, 리피터가 있다.
  - 데이터를 단지 전기적인 신호로 변환하여 주고받는 기능만 수행함.

2. Datalink Layer(데이터 링크 계층)
- 정의:
  - 물리계층을 통해 송수신되는 정보의 오류와 흐름을 관리하여 안전한 정보의 전달을 수행할 수 있도록 도와주는 역할을 하는 계층이다.
- 특징:
  - 통신 데이터의 단위는 frame
  - hop-to-hop 전송을 함.
  - MAC 주소를 가지고 통신을 한다.
  - 대표적인 장비는 스위치, 브릿지가 있다.
  - 대표적인 프로토콜은 HDLC, ADCCP, LLC, ALOHA

3. Network Layer(네트워크 계층)
- 정의:
  - 경로를 선택하고 주소를 정하고 경로에 따라 패킷을 전달해주는 역할을 하는 계층이다.
- 특징:
  - 통신 데이터의 단위는 packet
  - 라우팅, 흐름 제어, 세그멘테이션 등의 기능을 수행한다.
  - 대표적인 장비로는 라우터, 스위치가 있다.
  - 대표적인 프로토콜은 IP, ICMP, RIP

4. Transport Layer(전송 계층)
- 정의:
  - 데이터 전송의 오류 검출, 수정 및 흐름 제어를 수행하는 계층이다.
- 특징:
  - 통신 데이터의 단위는 segment
  - 데이터 전송을 위해 Port 번호를 사용한다.
  - 송신자와 수신자 간의 신뢰성있고 효율적인 데이터를 전송하기 위하여 오류검출 및 복구, 흐름제어와 중복검사 등을 수행
  - 대표적인 프로토콜은 TCP, UDP, SCTP

5. Session Layer(세션 계층)
- 정의:
  - 응용 프로그램 간의 대화를 유지하기 위한 구조를 제공하고, 이를 처리하기 위해 프로세스들의 논리적인 연결을 담당하는 계층이다.
- 특징:
  - 통신 중 연결이 끊어지지 않도록 유지시켜주는 역할 수행하기 위해 TCP/IP 세션 연결의 설정과 해제, 세션 메세지 전송 등의 기능을 수행한다.
  - 통신을 하기 위한 세션 확립, 유지, 중단을 수행한다.
  - 통신하는 사용자들을 동기화해주고, 오류 복구 명령들을 일괄적으로 처리한다.
  - 대표적인 프로토콜은 RPC, NetBIOS

6. Presentation Layer(표현 계층)
- 정의:
  - 애플리케이션이 다루는 정보를 통신에 알맞은 형태로 만들거나, 하위 계층에서 온 데이터를 사용자가 이해할 수 있는 형태로 만드는 역할을 담당하는 계층이다.
- 특징:
  - 데이터의 효율과 보안을 위해 압축과 암호화를 수행하고, 전송을 위한 포맷으로 변경을 수행한다.
  - 대표적인 프로토콜은 JPEG, MPEG이다.

7. Application Layer(응용 계층)
- 정의:
  - 응용 프로세스와 직접 관계하여 일반적인 응용 서비스를 수행하는 역할을 담당하는 계층이다.
- 특징:
  - 응용 프로세스 간의 정보 교환, 파일 전송 등의 서비스를 제공한다.
  - 대표적인 장비는 L7 스위치
  - 대표적인 프로토콜은 HTTP, FTP, SMTP, POP3


##### TCP/IP 모델
- TCP/IP 모델은 IETF라는 단체에서 제정하였다.
![image](/image/tcpip.png)
1. Network Interface(네트워크 인터페이스 계층)
- 정의:
  - 물리적인 네트워크 매체를 통해 데이터를 전송하는 기능을 담당하는 계층이다.
- 특징:
  - 이더넷, Wi-Fi 등 다양한 LAN 및 WAN 기술을 지원한다.
  - 하드웨어 주소(MAC 주소)를 사용하여 네트워크 장치를 식별한다.

2. Internet(인터넷 계층)
- 정의:
  - 패킷의 라우팅 및 주소 지정을 담당하고, 서로 다른 네트워크 간의 데이터 전송을 가능하게 하는 계층이다.
- 특징:
  - IP, ICMP, ARP 프로토콜을 지원한다.

3. Transport(전송 계층)
- 정의:
  - 애플리케이션 간의 통신을 관리하고, 흐름 제어 및 오류 검출 기능을 하는 계층이다.
- 특징:
  - TCP, UDP 프로토콜을 지원한다.

4. Application(응용 계층)
- 정의:
  - 사용자 요청을 처리하고, 데이터 전송을 위해 필요한 모든 기능을 수행하는 계층이다.
- 특징:
  - HTTP, FTP, SMTP 프로토콜을 지원한다.
  - 사용자 인터페이스와 가장 가까운 계층으로, 데이터의 형식을 정의한다.
  - 클라이언트-서버 모델을 기반으로 데이터 송수신을 처리한다.

#### 네트워크의 종류 (Types of Networks)

##### LAN(Local Area Network), WAN(Wide Area Network), MAN(Metropolitan Area Network), PAN(Personal Area Network)
![image](/image/network.png)
1. LAN(Local Area Network)
- 정의:
  - 근거리 통신망으로, 지역적인 범위에서 컴퓨터와 기타 장치들이 서로 연결되어 데이터를 공유하는 네트워크이다.
- 특징:
  - 조직내부나 동일 건물등 비교적 좁은 지역을 연결하기 위해 사용한다.
  - 초기 투자비용이 많이들고 유지비용은 적게든다.
  - 관리자가 직접 관리하는 방식

2. WAN(Wide Area Network)
- 정의:
  - 넓은 지리적 범위(도시, 국가, 대륙 등)에 걸쳐 여러 LAN을 연결하는 네트워크이다.
- 특징:
  - 일반적으로 공공 통신 인프라를 통해 데이터 전송을 수행한다.
  - 수백 킬로미터에서 수천 킬로미터까지의 범위를 커버한다.
  - 위성, 전화선, 광섬유 등 다양한 전송 매체를 사용하여 연결된다.
  - 설치 및 운영 비용이 상대적으로 높다.
- 예시:
  - 기업의 자사 간 연결
  - 국가 및 국제 데이터 센터 간 연결

3. MAN(Metropolitan Area Network)
- 정의:
  - 특정 도시나 대도시 지역 내에서 여러 LAN을 연결하는 네트워크이다.
- 특징:
  - 일반적으로 수십킬로미터 범위 내에서 운영된다.
  - WAN보다 좁고, LAN보다 넓은 범위를 커버한다.
  - WAN보다 빠른 데이터 전송 속도를 제공한다.
- 예시:
  - 도시 내의 기업 및 기관 간 연결
  - 대학 캠퍼스 네트워크
  - 공공 서비스 및 정부 기관의 네트워크

4. PAN(Personal Area Network)
- 정의:
  - 개인의 장치 간에 데이터를 전송하기 위해 사용되는 소규모 네트워크이다.
- 특징:
  - 일반적으로 개인이 소지한 장치 간의 연결을 의미한다.
  - 10~30m 정도의 짧은 범위를 커버한다.
  - 주로 Bluetooth, Wi-Fi Direct, Zigbee 등이 사용된다.
- 예시:
  - 개인 기기 간 데이터 공유
  - IoT 장치 간의 통신
  - 블루투스 스피커와 스마트폰의 연결

##### WLAN(Wireless LAN), SAN(Storage Area Network)
1. WLAN(Wireless LAN)
- 정의:
  - 유선 연결 대신 무선 기술을 사용하여 로컬 네트워크 노드를 연결하는 근거리 통신망(LAN)의 한 유형이다.
- 특징:
  - 무선 주파수(RF) 또는 적외선 기술을 사용하여 데이터를 공중으로 전송하고 수신한다.
  - 케이블 없이 디바이스를 네트워크에 연결한다.
  - 설치, 유지보수, 재배치가 간편하다.
  - 액세스 포인트(AP)를 이용하여 신호를 증폭하고 인터넷에 연결한다.
- 예시:
  - 가정용 Wi-Fi 네트워크
  - 사무실 무선 네트워크
  - 공공 장소의 Wi-Fi 핫스팟 (카페, 호텔, 공항 등)

2.SAN(Storage Area Network)
- 정의:
  - 여러 컴퓨터와 서버가 공유할 수 있는 고속 전용 스토리지 네트워크이다.
- 특징:
  - 일반 네트워크와 분리된 전용 스토리지 네트워크로 동작한다.
  - 블록 단위 데이터 전송을 지원하여 빠르고 안정적인 데이터 접근을 제공한다.
  - 스토리지 디바이스들이 네트워크로 연결되어 중앙화된 논리 볼륨을 형성한다.
- 예시:
  - 데이터베이스 서버
  - 가상화 배포
  - 대규모 가상 데스크톱 인프라

#### 네트워크 토폴로지 (Network Topology)
- 정의:
  - 네트워크를 배치하는 방식으로, 링크와 노드 들이 어떤 식으로 배치되어 서로 연결되는지를 물리적으로 혹은 논리적으로 설명하는 것이다.
##### 스타(Star), 버스(Bus), 링(Ring), 메시(Mesh), 트리(Tree)
1. 스타(Star)
- 정의:
  - 가장 흔하게 쓰이는 네트워크 토폴로지로, 네트워크 안의 모든 노드가 동축 케이블이나 연선 또는 광케이블을 통해 직접 하나의 중앙 허브에 연결된 형식이다.
- 특징:
![image](/image/star-topology.png)
  - 중앙 노드는 서버로서 작동하여 데이터의 흐름을 관리합니다.
  - 각각의 노드에서 보낸 정보가 목적지에 닿기 위해서는 반드시 중앙 노드를 지나야 한다.
  - 중앙 노드는 리피터처럼 데이터 손실을 막아준다.
- 장점:
  - 한 장소에서 전체 네트워크를 관리할 수 있다.
  - 각 노드는 독립적으로 연결되어 있어, 노드 하나의 장애가 발생해도 다른 노드는 운용 가능하다.
  - 장애가 발생한 경우 원인을 쉽게 찾을 수 있다.
- 단점:
  - 중앙 허브에 문제가 생기면 전체 네트워크가 모두 멈춘다.
  - 전체 네트워크의 성능이 중앙 노드에 의해 결정된다.

- 예시:
  - 가정용 네트워크
  - 소규모 사무실 네트워크

2. 버스(Bus)
- 정의:
  - 네트워크상의 모든 장치가 하나의 케이블로 연결되어 있는 유형이다.
- 특징:
![image](/image/bus-topology.png)
  - 네트워크의 한쪽 끝에서 다른 쪽 끝으로 한 방향으로 연결되어 있기에 “선형 토폴로지" 또는 “백본 토폴로지"라고도 지칭한다.
  - 데이터의 흐름이 케이블의 경로를 따라 한 방향으로 이동한다.
- 장점:
  - 비용이 효율적이고, 소규모 네트워크에 적합하다.
  - 장치의 추가, 제거가 쉽다.
- 단점:
  - 케이블에 장애가 발생할 경우, 모든 네트워크가 중단된다.
  - 노드를 추가할 때마다, 대역폭이 낭비된다.
  - 양 방향 통신이 불가능하다.
- 예시:
  - 이벤트, 전시회 등의 임시 네트워크


3. 링(Ring)
- 정의:
  - 노드들이 원 또는 고리 모양으로 배열되어 있는 유형이다.
- 특징: 

    ![image](/image/ring-topology.png)
  - 데이터가 원 모양의 네트워크를 따라 한 방향 또는 양방향으로 흐른다.
  - 각 노드의 양 옆에는 두 개의 이웃 노드가 존재한다.
- 장점:
  - 한 번에 하나의 노드에서만 데이터를 전송할 수 있어 패킷 충돌의 위험이 낮다.
  - 설치 비용이 낮고, 문제 발생 시 원인 파악이 쉽다.
- 단점:
  - 한 노드에 문제 발생 시 모든 네트워크가 중단된다.
  - 노드 추가, 제거 시에 전체 네트워크가 중단된다.
- 예시:
  - LAN에서 주로 사용한다.

4. 매시(Mesh)
- 정의:
  -  노드들을 점 대 점으로 상호 연결한 구조로 복잡하고 정교한 유형이다.
- 특징:

    ![image](/image/mesh-topology.png)
  - 모든 노드가 상호연결되어 있다.
  - 라우팅 방식으로 전송 시, 최단 거리를 결정하여 데이터를 전송한다.
  - 플러딩 방식으로 전송 시, 네트워크 내부의 모든 노드에 데이터를 전송한다.
- 장점:
  - 안정성과 보안성이 높다.
  - 노드 간에 상호 연결된 정도가 높고 복합적이어서 하나의 노드에 문제가 발생해도, 나머지는 운용 가능하다.
- 단점:
  - 네트워크 관리, 설치, 재구성이 어렵다.
  - 케이블 등 설치 비용이 많이 든다.
- 예시:
  - WAN에서 주로 사용되고, 군사용 네트워크 등 안정한 통신이 필요한 분야에서 사용.
  - IoT, 블록체인 네트워크 구성

5. 트리(Tree)
- 정의:
  - 노드들이 부모-자식 계층구조로 연결된 유형이다.
- 특징:

    ![image](/image/treetopology.png)
  - 모든 노드가 상호연결되어 있다.

- 장점:
  - 네트워크의 확장이 매우 쉽다.
  - 성능에 문제가 생긴 경우, 각 노드에 개별적으로 접근할 수 있어 문제 해결이 쉽다.
- 단점:
  - 상위 계층의 장치에 문제 발생 시, 하위 계층의 장치도 영향을 받을 수 있다.
  - 구조가 복잡하다.
- 예시:
  - 대규모 기업 네트워크
  - 캠퍼스 네트워크

## 데이터 통신 및 전송 기술

#### 데이터 표현과 신호 (Data Representation and Signals)

##### 디지털 데이터 - 이진 신호, 샘플링
1. 이진 신호(Binary Signal)
- 정의:
  - 0과 1로 이루어진 비연속적이고, 이산 시간 신호이다.
- 특징:
  - 이진 형식으로 데이터를 전송한다.
  - 노이즈에 대한 저항력이 강해 오류 발생 가능성이 낮다.
  - 컴퓨터와 디지털 장치에서 쉽게 처리할 수 있다.

2. 샘플링
- 정의:
  - 아날로그 신호의 연속적인 값을 일정한 시간 간격으로 추출하여 이산적인 값으로 변환하는 과정이다.
- 특징:
  - 원래의 데이터에서 샘플을 추출하여, 추출한 데이터만 사용하는 방법이다.
##### 아날로그 신호 - 주파수, 진폭, 위상
1. 주파수
- 정의:
  - 신호가 단위 시간동안 반복되는 횟수를 나타내는 물리량이다.
- 특징:
  - 일반적으로 단위는 Hz를 사용한다.
  - 주파수가 높을 수록 주기적인 현상이 빈번하게 발생한다.

2. 진폭
- 정의:
  - 신호의 최대 변동 범위를 나타낸다.
- 특징:
  - 신호의 강도나 세기를 의미한다.
  - 아날로그 신호에서는 주로 전압의 크기로 표현한다.
  - 진폭이 클수록 신호의 강도가 강하다.

3. 위상
- 정의:
  - 주기적인 신호의 특정 시점에서의 위치를 나타내는 각도이다.
- 특징:
  - 주로 라디안이나 도(degree) 단위로 표현된다.
  - 서로 다른 위상을 가진 신호가 겹칠 경우, 간섭 현상이 발생할 수 있다.

#### 부호화와 변조 (Encoding and Modulation)
1. 부호화(Encoding)
- 정의:
  - 데이터를 다른 형식으로 변환하는 과정이다.
- 특징:
  - 입력한 문자나 기호를 컴퓨터가 읽고 처리할 수 있는 형태로 변환한다.
  - 데이터를 효율적으로 전송하거나 저장하기 위한 목적으로 사용한다.
- 예시:
  - 문자 인코딩(UTF-8, ASCII)
  - URL 인코딩(URL에서 사용될 수 없는 특수 문자는 %에 숫자를 붙여 인코딩한다.)

2. 변조(Modulation)
- 정의:
  - 인코딩(부호화)된 데이터를 원래의 형식으로 변환하는 과정이다.
- 특징:
  - 데이터를 복원하여 사람이 이해하고 처리할 수 있는 형태로 변환하는 작업을 한다.
  - 이진수 데이터를 사람이 처리할 수 있는 문자로 변환한다.
- 예시:
  - ASCII 디코딩(65 -> 'A')
  - Base64 디코딩(SGVsbG8sIFdvcmxkIQ== -> "Hello, World")

##### 라인 부호화(Line Coding), 주파수 변조(FM), 진폭 변조(AM), 위상 변조(PM)
1. 라인 부호화(Line Coding)
- 정의:
  - 디지털 데이터를 디지털 신호로 바꾸는 것이다.
- 특징:
  - 이진 데이터를 전기 신호로 변환하여 전송한다.
  - 연속적인 신호에서 비트를 구분할 수 있도록 설계해야, 수신측에서 정확하게 해석할 수 있다.
  - 대칭적인 신호 형태를 제공하여, 신호의 전송 과정에서 발생할 수 있는 왜곡이나 변형을 최소화한다.
- 예시:
  - NRZ, RZ, Manchester Encoding

2. FM(Frequency Modulation, 주파수 변조)
- 정의:
  - 정보 신호의 주파수를 변조하여 전송하는 방식이다.
- 특징:
  - 잡음에 대한 저항력이 강해 신호 품질이 우수하다.
  - FM은 넓은 대역폭을 필요로 하며, 나이퀴스트 정리에 따라 최대 주파수의 두 배 이상이 필요하다.
- 예시:
  - 라디오, TV, 무선 통신


3. AM(Amplitude Modulation, 진폭 변조)
- 정의:
  - 정보 신호의 진폭을 변조하여 전송하는 방식이다.
- 특징:
  - 아날로그 신호의 진폭이 변할 때, 그에 따라 반송파의 진폭도 변한다.
  - AM 신호는 간섭과 잡음에 취약하여 신호 품질이 저하될 수 있다.
  - AM은 대역폭이 넓어, 두 배의 변조 주파수 대역을 필요로 한다.
- 예시:
  - 라디오

4. PM (Phase Modulation, 위상 변조)
- 정의:
  - 정보 신호의 위상을 변조하여 전송하는 방식이다.
- 특징:
  - PM은 FM과 유사하게 잡음에 대한 저항력이 강하다.
  - 주파수 변조와 마찬가지로 나이퀴스트 정리에 따른 고려가 필요하다.
- 예시:
  - QPSK(Quadrature Phase Shift Keying)

##### 다중화 기술 - TDM(Time Division Multiplexing), FDM(Frequency Division Multiplexing), CDM(Code Division Multiplexing)
1. TDM (Time Division Multiplexing, 시간 분할 다중화)
- 정의:
  - 한 전송로에 주어진 시간대역을 여러 타임슬롯으로 분할하여 각 채널에 배당함으로써 많은 채널이 하나의 전송회선을 공유하는 방식이다.
- 특징:
  ![image](/image/tdm.png)
  - 채널간 간섭을 막기 위해 보호시간 필요
  - 여러 다른 채널로부터 정보를 샘플링하여 채널 사이에 시차를 두어 하나의 회선으로 다수의 채널을 전송한다.


2. FDM (Frequency Division Multiplexing, 주파수 분할 다중화)
- 정의:
  - 전송되어야 하는 신호들의 대역폭을 합한 것보다 링크의 대역폭이 클 때 적용할 수 있는 아날로그 기술로, 한 전송로의 대역폭을 여러 개의 작은 채널로 분할하여 여러 단말기가 동시에 이용하는 방식이다.
- 특징:
  ![image](/image/fdm.png)
  - 전송되어야 하는 신호들의 대역폭을 합한 것보다 링크의 대역폭이 클 때 적용할 수 있는 아날로그 기술이다.
  -  각 장치로부터 받은 신호를 각기 다른 반송 주파수로 변조하여, 링크를 통해 이동할 수 있는 하나의 복합신호로 합쳐지게 된다.


3. CDM (Code Division Multiplexing, 코드 분할 다중화)
- 정의:
  - 여러 신호를 서로 다른 코드로 인코딩하여 동시에 전송하는 방식이다
- 특징:
  - 모든 신호가 동일한 주파수 대역에서 동시에 전송되지만, 각 신호는 고유한 코드로 구분된다.
  - 코드의 특성으로 인해 잡음에 대한 저항력이 강하다.
  - FDM과 TDM의 혼합 방식이다.
- 예시:
  - CDMA와 같은 무선 통신 시스템에서 널리 사용된다.


#### 전송 방식 (Transmission Methods)	

##### 직렬 전송(Serial Transmission), 병렬 전송(Parallel Transmission)
1. 직렬 전송(Serial Transmission)
- 정의:
  - 데이터를 한 비트씩 순차적으로 전송하는 방식이다.
- 특징:
  - 통신 회선을 통하여 한 번에 한 비트씩 순서대로 전송하는 방식
  - 전송속도는 느리지만, 통신 회선의 비용이 저렴하다.
- 예시:
  - 이더넷, HDMI 등의 원거리 통신

2. 병렬 전송(Parallel Transmission)
- 정의:
  - 별렬로 나열하여, 한 번에 n비트 이상의 데이터를 전송하는 방식이다.
- 특징:
  - 여러 개의 전송로를 통하여 동시에 여러 비트를 전송한다.
  - 직렬 전송에 비해 속도는  빠르나, 통신회선을 구축하는 데 많은 비용이 소요된다.
- 예시:
  - IC회로, RAM 등 근거리 통신에서 사용한다.

## 네트워크 프로토콜

#### 프로토콜의 개념과 역할 (Concept and Role of Protocols)	
- 정의:
  - 컴퓨터 네트워크에서 장치 간 데이터를 주고 받는 데 사용되는 합의된 규칙/규약
- 역할:
  - 데이터가 어떻게 구조화되고 인코딩되는지를 정의하여, 수신자가 데이터를 올바르게 해석할 수 있도록 한다.
  - 데이터가 전송되는 방식(예: 신뢰성 있는 전송, 비신뢰성 전송 등)을 규정하여, 효율적인 데이터 흐름을 보장한다.
  - 전송 중 발생할 수 있는 오류를 감지하고 수정하는 방법을 정의하여, 데이터의 무결성을 유지한다.
#### 데이터 링크 계층 프로토콜 (Data Link Layer Protocols)

##### 이더넷(Ethernet), PPP(Point-to-Point Protocol), HDLC(High-Level Data Link Control)
1. 이더넷(Ethernet)
- 정의:
  - 컴퓨터 네트워크 간의 데이터 통신 기술로, 주로 LAN에서 사용되는 프로토콜이다.
- 특징:
  - 허브, 스위치 등 같은 장비에 연결된 같은 네트워크 내부에서의 데이터 전송에서 사용한다.
  - 데이터 링크 계층에서 MAC 주소와 프로토콜의 형식을 정의한다.
  - 광섬유 케이블, 구리선 등의 물리적 매체를 통해 데이터를 전송할 수 있다.
  - CSMA/CD 방식을 사용하여, 네트워크에서 여러 장비가 동시에 데이터를 전송할 때 발생할 수 있는 충돌을 감지한다.
- frame 구조:
  ![image](/image/ethernetframe.png)
  https://velog.io/@moonblue/%EC%9D%B4%EB%8D%94%EB%84%B7-%ED%94%84%EB%A0%88%EC%9E%84-Ethernet-Frame
  - Preamble: 프레임의 시작을 알리는 동기화 비트 패턴으로, 수신 측에서 프레임의 시작을 인식할 수 있게한다.
  - Destination address: 수신할 장치의 MAC 주소를 나타낸다.
  - Source address: 데이터를 전송한 장치의 MAC 주소를 나타낸다.
  - Type: 프레임 내의 페이로드 데이터 형식을 식별하는 필드로, 상위 계층 프로토콜(예: IPv4, IPv6)의 종류를 알려준다.
  - FCS(Frame Check Sequence): 프레임의 무결성을 확인하기 위한 오류 검출 코드로, 수신 측에서 프레임의 오류를 검출한다.

2. PPP(Point-to-Point Protocol)
- 정의:
  - 두 지점 간의 데이터 통신을 위한 프로토콜이다.
- 특징:
  - HDLC 프로토콜의 프레임에 프로토콜 필드를 추가한 프레임 구조를 가진다.
  - IP 등 여러 상위 프로토콜을 지원할 수 있게 설계됨.
  - FCS를 이용하여 데이터 전송 중 발생할 수 있는 오류를 검출한다.
- frame 구조:
  ![image](/image/ppp.png)
  - 앞, 뒤 필드에는 flag값으로 0x7e 값을 가지고, 처음과 끝을 의미한다.
  - 주소 필드에는 하나의 데이터링크에서 사용되므로 0xFF로 설정한다.
  - 제어 필드에는 0x03으로 비연결형 프레임을 나타낸다.
  - 프로토콜 필드에는 IP, LCP, NCP, PAP, CHAP를 구분하는 필드이다.
  - CRC 필드에는 checksum을 통해 오류를 감지하는 필드이다.

##### MAC(Media Access Control)과 프레임 처리
- 정의:
  - 네트워크 상의 장치들이 공유 매체에 접근할 때 충돌을 방지하고 효율적으로 통신할 수 있도록 하는 프로토콜이다.
- 특징:
  - 각 네트워크 장치는 고유한 MAC 주소를 가지고 있으며, 이를 통해 장치 간의 통신이 가능하다.
  - 여러 장치가 동시에 데이터를 전송할 경우 발생할 수 있는 충돌을 방지하는 규칙을 제공한다. 이더넷에서는 CSMA/CD 방식 사용한다.
  - MAC 계층은 데이터 프레임을 생성하고 전송하며, 수신된 프레임의 유효성을 검사한다.
  - 데이터가 전송될 때, 송신자의 프레임에는 송,수신자의 MAC 주소를 포함한다.
  - 데이터가 수신될 때, 수신자는 자신의 MAC 주소와 프레임의 MAC 주소를 비교하여 일치할 때만 데이터를 수신하고 추출하여 상위 계층으로 전달한다.

#### 네트워크 계층 프로토콜 (Network Layer Protocols)
##### IPv4와 IPv6 - 구조, 헤더, 주소 체계
- IP 프로토콜은 TCP/IP 프로토콜이 사용하는 전송 메커니즘이다.
1. IPv4
- 정의:
  - 인터넷 프로토콜의 네 번째 버전으로, 32비트의 주소를 사용하는 프로토콜이다.
- 특징:
  - 32비트 길이의 주소로, 총 2^32개의 주소를 가지고 있다.
  - 8비트씩 4자리로 표현하고, 각 자리는 0~255까지의 값을 가진다.
  - 전체 주소를 클래스 A,B,C,D,E로 나누어 클래스 별로 구간을 가지고, 용도에 따라 배분한다.
  - 비연결형 프로토콜로, 데이터그램 전송 시 연결을 설정하지 않는다.
  - 오류 처리가 포함되지 않아, 상위 계층에서 이 기능을 담당한다.
  - 주소의 수가 한정되어 있어, IPv6로의 전환이 논의되고 있다.
- IP Datagram 헤더 구조
  ![image](/image/ipheader.png)
  - VER: IPv4, IPv6 중 IP 버전을 나타낸다.
  - HLEN: IP 헤더의 길이를 나타낸다. 4바이트 단위로 계산된다.
  - Service Type: 데이터그램의 우선순위와 라우터에 요구되는 서비스종류가 들어가는 부분이다.
  - Total Length: IP 데이터그램 전체 길이를 나타낸다. 
  - Identification: 데이터그램을 식별하기 위한 고유값이다.
  - Flags: 데이터그램의 분할(프래그먼트) 여부를 제어하는 플래그 비트이다. DF, MF로 구분된다.
  - Fragmentation Offset: 데이터그램 조각의 원본 데이터에서의 위치를 나타낸다.
  - TTL (Time to Live): 데이터그램의 유효 시간을 나타낸다. (라우터를 지날 때마다 1씩 감소)
  - Protocol: 상위 계층 프로토콜을 지정한다. (예: TCP=6, UDP=17)
  - Header Checksum: 헤더의 무결성을 확인하는 체크섬 값이다.
  - Source Address: 송신자의 IP 주소를 나타낸다.
  - Destination Address: 수신자의 IP 주소를 나타낸다.
  - Padding: 헤더를 32비트 단위로 정렬하기 위해 추가되는 값이다.
- 주소 체계
  - 8비트씩 4개로 나누어 점으로 구분하여 표현한다.
  - 전체 주소는 네트워크 부분과 호스트 부분으로 나뉜다.
  - 클래스 A는 0.0.0.0 ~ 127.255.255.255
  - 클래스 B는 128.0.0.0 ~ 191.255.255.255
  - 클래스 C는 192.0.0.0 ~ 223.255.255.255
  - 클래스 D는 224.0.0.0 ~ 239.255.255.255
  - 클래스 E는 240.0.0.0 ~ 255.255.255.255

##### ICMP(Internet Control Message Protocol), ARP(Address Resolution Protocol), RARP(Reverse ARP)
1. ICMP(Internet Control Message Protocol)
- 정의:
  -  디바이스에서 네트워크 통신 문제를 진단하는 데 사용하는 프로토콜이다.
- 특징:
  - IP 프로토콜에섯 오류 검사와 오류 제어를 하지 않아 설계되었다.
  - 단지 오류 보고만 하고, 수정은 상위 계층 프로토콜에서 처리한다.
  - 오류 보고 메시지는 라우터 또는 목적지 호스트에서 생성된다.
  - 수신은 오직 발신지 호스트에서만 가능하다.
  - 오류 메시지에 대한 오류 메시지는 생성되지 않는다.
- ICMP 데이터그램 헤더 구조
  ![image](/image/icmp.png)
  - Type: 패킷 내부에 어떤 종류의 ICMP 메시지가 있는 지 정의한다.
  - Code: Type에 대한 설명이 들어가 있다.
  - Checksum: 전달된 패킷이 이상없이 잘 도착하였는지를 체크한다.
- 5가지의 감지 오류 유형
- 목적지 도달 불가 메시지 (Destination Unreachable Message)
  - 라우터나 호스트가 데이터그램을 전달할 수 없을 때 생성
- 발신지 억제 메시지 (Source Quench Message)
  - 데이터그램이 폐기되었거나, 송신 경로 상에 혼잡이 발생했음을 알릴 때 생성
- 매개변수 문제 메시지 (Parameter Problem Message)
  - 데이터그램 필드에서 불명확하거나, 빠진 값을 발견할 때 생성
- 재지정 메시지 (Redirection Message)
  - 잘못된 라우터에게 데이터그램이 전송될 때 생성

2. ARP(Address Resolution Protocol)
- 정의:
  - 네트워크에서 IP 주소를 물리적 주소(일반적으로 MAC 주소)로 변환하는 데 사용되는 프로토콜이다.
- 특징:
  - ARP request 은 브로드캐스트로, ARP reply는 유니캐스트로 전송한다.
  - IP 주소를 이용하여 MAC 주소를 찾기 위해 사용한다.
  - 동일 네트워크 내에서만 작동한다.
  - IPv4에서만 사용한다.

- ARP packet 구조
  ![image](/image/arp.png)
  - Hardware Type: ARP가 수행되고 있는 네트워크 유형을 정의
  - Protocol Type: 상위 프로토콜을 정의
  - Hardware Length: 물리주소의 길이를 바이트 단위로 정의
  - Protocol Length: 논리주소의 길이를 바이트단위로 정의
  - Operation: 패킷의 유형을 정의
  - Sender Hardware Address: 송신자의 물리주소를 나타내는 필드
  - Sender IP Address: 송신자의 논리주소를 나타내는 필드
  - Target Hardware address: 목적지 시스템의 물리주소를 정의
  - Target IP Address: 목적지 시스템의 논리주소를 정의

- ARP 동작 과정
  ![image](/image/arprequest.png)
  ![image](/image/arpreply.png)
  - System A에서, IP 주소 141.23.56.23을 가진 다른 시스템 B에게 보낼 패킷을 가지고 있음.
  - 물리 주소를 모르기 떄문에, ARP request packet을 브로드캐스트로 전송.
  - System B에서 자신의 물리 주소를 포함한 ARP reply packet을 System A에게 전송
  - 이후 A와 B는 데이터 송신 가능

3. RARP(Reverse ARP)
- 정의:
  - ARP 프로토콜과 반대로 MAC 주소로 IP값을 알아내는 프로토콜이다.
- 특징:
  - MAC 주소는 알지만 IP 주소를 모를 때 사용한다.
  - ARP와 반대로 MAC 주소를 담은 패킷을 브로드캐스트로 전송하고, IP 주소를 담은 패킷을 유니캐스트로 받는다.

#### 전송 계층 프로토콜 (Transport Layer Protocols)

##### TCP(Transmission Control Protocol), UDP(User Datagram Protocol)
1. TCP(Transmission Control Protocol)
- 정의:
  - 데이터를 전송하기 전에 먼저 두 호스트(통신 장치 또는 종단 시스템) 사이에 논리적 연결을 설정하는 신뢰성 있는 연결 지향(connection-oriented) 프로토콜이다.
- 특징:
  - 연결 지향 방식으로 세그먼트를 교환한다.
  - 3-way handshaking 과정을 통해 연결을 설정한다.
  - 흐름 제어 및 혼잡 제어.
  - 전이중(Full-Duplex), 점대점(Point to Point) 방식.
    - 흐름 제어:
      - 목적지로부터 확인응답(ACK)을 수신하기 전에 발신지가 전송할 수 있는 데이터의 양을 정의한다.
      - 데이터는 송신 버퍼를 통해 전달하고, 데이터의 크기는 슬라이딩 윈도우 프로토콜로 윈도우 크기를 정의하여 결정한다.
    - 혼잡 제어
      - 혼잡 윈도우(Congestion Window, cwnd)의 크기를 축소, 확대시키면서 혼잡에 대응한다.
      - 중복 ACK가 3번 수신되거나 RTT가 급격히 증가했을 때, 혼잡이라고 판단한다.
      - 초기에는 cwnd를 작은 크기에서 시작하고, 점진적으로 크기를 늘려나간다.
    - 연결 설정, 데이터 전송, 연결 종료의 과정으로 데이터를 주고 받는다.
- 연결 설정:
  ![image](/image/tcpconnect.png)
  - 3-Way Handshaking 
  - Client에서 SYN 세그먼트를 server에 전송한다.
  - Server에서는 SYN을 받았다는 응답과 초기 수신 버퍼의 크기를 담아, SYN과 ACK 세그먼트를 Client에 전송한다.
  - Client에서는 SYN과 ACK를 수신하고, ACK를 다시 전송한다. 이후 connection을 연다.
  - Server에서는 ACK를 수신하고 connection을 연다.
- 예시:
  - 데이터의 순서와 신뢰성이 중요한 금융 거래 시스템이나 데이터베이스 관리 시스템

1. UDP(User Datagram Protocol)
- 정의:
  - 데이터를 데이터그램 단위로 처리하는 프로토콜이다.
- 특징:
  - 비연결형 서비스로 데이터그램 방식을 제공한다.
  - 정보를 주고 받을 때 정보를 보내거나 받는다는 신호절차를 거치지 않는다.
  - UDP헤더의 CheckSum 필드를 통해 최소한의 오류만 검출한다.
- 예시:
  - 실시간 스트리밍 서비스

##### 포트 번호와 소켓 프로그래밍(Socket Programming)
1. 포트 번호
- 정의:
  - TCP/IP 네트워크에서 특정 프로세스나 서비스를 식별하기 위해 사용되는 숫자이다.
- 특징:
  - 포트 번호는 0 ~ 65,535 사이의 번호이다.
  - well-known port number는 0 ~ 1023, temporary port number는 49,152 ~ 65,535 사이의 번호이다.
  - 서버의 경우 주로 well-known port number를 사용하고, 클라이언트일 경우 주로 temporary port number를 사용한다.
  
2. 소켓 프로그래밍(Socket Programming)
- 정의:
  - 컴퓨터 네트워크를 통해 데이터를 송수신하기 위해 소켓을 사용하는 프로그래밍 기술이다.
- 특징:
  - 통신 연결 요청 종류에 따라 Client, Server로 각각 나뉜다.
  - 
- 소켓 통신 구조
  ![image](/image/socket.png)
  - Client
    - 소켓을 생성한다.
    - Server에 연결을 요청한다.
    - Server에서 연결을 수락하면 데이터를 송수신한다.
    - 데이터 교환이 완료되면 소켓을 닫는다.
  - Server
    - 소켓을 생성한다.
    - Server의 IP주소와 포트 번호를 소켓에 bind한다.
    - Client로부터의 요청을 기다린다.
    - 요청이 들어오면 accept하고, 새로운 소켓을 생성한다.
    - Client와 연결 후, 데이터를 송수신한다.
    - 교환이 완료되면 소켓을 닫는다.

#### 응용 계층 프로토콜 (Application Layer Protocols)

##### HTTP/HTTPS(Web Communication)
- 정의:
  - 웹에서 데이터를 주고받는 서버-클라이언트 모델의 프로토콜이다.
- 특징:
  - 클라이언트와 서버 간 요청과 응답이 이루어진 후, 연결을 바로 종료한다.
  - 매 요청마다 새로운 연결을 생성해야 한다.
  - HTTP는 상태를 유지하지 않아서, 각 요청은 독립적이며 이전 요청과의 관계를 기억하지 않는다.
  - HTTP 메시지는 사람이 읽을 수 있는 텍스트 형식으로 작성된다.

##### FTP(File Transfer Protocol), SMTP/POP3/IMAP(Email Protocols)
1. FTP(File Transfer Protocol)
- 정의:
  - 파일을 이동할 때 사용하는 프로토콜
- 특징:
  - FTP 연결을 위해 데이터 포트와 제어 포트가 필요하다.
    - 파일 데이터 전송을 위한 데이터 포트는 20번 포트에서 사용한다.
    - 명령과 응답 등의 제어 정보를 포함한 제어 포트는 21번 포트에서 사용한다.
  - 연결 방식에는 Active/Passive Mode가 있다.
    - Active 모드는 서버가 클라이언트의 포트를 접속하여 전송하는 것이다.
    - Passive 모드는 클라이언트가 서버의 포트에 접속하여 데이터를 전송하는 것이다.
  - 장점:
    - 간단한 파일 전송에 특화
    - 대용량의 파일 안정적으로 전송
  - 단점:
    - 기본 FTP는 데이터와 비밀번호를 평문으로 전송하여 보안에 취약하다.

2. SMTP/POP3/IMAP(Email Protocols)
- 정의:
  - 이메일을 전송하기 위한 프로토콜이다.
- 특징:
  - SMTP는 이메일 발송, POP3는 이메일 수신 및 다운로드, IMAP는 이메일을 서버와 동기화하여 수신하는 목적으로 사용된다.
  - SMTP는 SSL, TLS를 사용하여 이메일을 전송의 보안성을 보장한다.
##### DNS(Domain Name System), DHCP(Dynamic Host Configuration Protocol)
1. DNS(Domain Name System)
- 정의:
  - 인터넷에서 도메인 이름을 IP주소로 변환하거나 반대로 IP주소를 도메인 이름으로 변환하는 네트워크 시스템이다.
- 특징:
  - 사람들은 숫자로된 IP 주소를 기억하지 않고, 문자로 된 주소만 기억하면 된다.
  - DNS는 전 세계에 분산된 여러 DNS 서버들에 의해 유지되는 분산 데이터베이스를 기반으로 동작한다.
  - DNS는 잘못된 도메인 이름이나 IP 주소를 입력한 경우에도 적절한 오류 처리를 지원한다.
  -  
- 작동 원리:
  ![image](/image/dns.png)
  - 사용자가 브라우저 주소창에 URL을 입력합니다.
  - 도메인의 IP주소를 알아내기 위해 DNS Resolver에 요청을 보낸다.
  - DNS Resolver는 로컬 DNS 캐시를 확인하고, 없으면 루트 DNS 서버에서 최상위 도메인(TLD) DNS서버의 IP주소를 알아낸다.
  - DNS Resolver는 TLD DNS 서버에 연결하여, 권한 네임 서버의 IP주소를 알아낸다. 
  - 이후 권한 네임 서버에서 최종 IP 주소롤 알아내서, 브라우저로 전달한다.

2. DHCP(Dynamic Host Configuration Protocol)
- 정의:
  - DHCP는 IP 주소 및 기타 통신 매개변수를 네트워크에 연결된 장치에 자동으로 할당한다.
- 특징:
  - 대부분의 가정용 네트워크에서는 라우터가 IP 주소를 장치에 할당하는 DHCP 서버의 역할을 한다.
  - PC의 수가 많거나 변동 사항이 많은 경우 IP 설정이 자동으로 되어 효율적이고 IP충돌을 막을 수 있다.
  - DHCP 서버가 다운되면 IP 할당이 제대로 이루어지지 않는다.
- 작동 원리:
  - 장치가 네트워크에 연결 시 IP주소를 요청한다.
  - DHCP 서버로 요청이 전달되고, IP 주소를 할당한다.
  - 장치의 연결이 해제되거나 시간이 지나면, IP 할당을 해제한다.
- 주소 할당 방식:
  - 동적 할당(Dynamic Allocation)
    - 관리자가 DHCP에 IP주소를 자동으로 설정하도록 설정한다.
  - 자동 할당(Automatic allocation)
    - DHCP 서버는 관리자가 정한 규칙에 따라 IP주소를 영구 할당한다.
  - 수동 할당(Manual allocation)
    - 관리자가 직접 클라이언트에 대한 고유 식별자를 IP주소에 할당한다.
## 네트워크 장비 (Networking Devices)

#### 스위치(Switch)와 허브(Hub)
1. 허브(Hub)
- 정의:
  - 전기적 신호를 증폭시켜 여러 대의 장비를 LAN에 접속할 수 있도록 하는 장비다.
- 특징:
  - 1개의 포트에 하나의 장치가 할당된다.
  - 주로 OSI 1계층에서 작동한다.
  - 허브로 연결된 네트워크에서, 하나의 장치에 전송된 Frame을 허브에 연결된 모든 장치에 전송할 경우 Flooding이 발생할 수 있다.
  - 보안성이 떨어진다.
  - 연결된 장치의 수에 따라 대역폭을 분리하여 제공한다.
- 사용 이유:
  - 멀티 포트 기능: 한 번에 많은 장비를 연결할 수 있다.
  - 리피터 기능: LAN의 최대 전송 거리를 길게 할 수 있다.

2. 스위치(Switch)
- 정의:
  - 허브와 다르게 정해진 목적지에만 데이터를 전송하는 네트워크 장치이다. 
- 특징:
  - MAC주소를 기반으로 특정 목적지에만 데이터를 전송한다.
  - 주로 OSI 2계층에서 작동한다.
  - 각 포트 별로 고유 대역폭을 사용하여 충돌이 발생하지 않는다.
  - MAC 주소와 포트 번호가 기록된 MAC 주소 테이블을 가지고 있다.
- 사용 이유:
  - 허브에 비해 정해진 목적지에 데이터를 전송하므로, 프레임 충돌이 감소하여 네트워크 성능이 향상된다.

#### 라우터(Router)와 게이트웨이(Gateway)
1. 라우터(Router)
- 정의:
  - 네트워크와 네트워크 간의 경로(Route)를 설정하고 가장 빠른 길로 트래픽을 이끌어주는 네트워크 장비입니다.
- 특징:
  - IP주소를 기반으로 패킷을 전송한다.
  - 주로 OSI 3계층에서 작동한다.
  - 라우팅 테이블을 이용하여 패킷을 올바른 경로로 전달한다.
  - NAT(Network Address Translation) 기능을 통해, 여러 장치가 공유 IP 주소를 사용하여 인터넷에 접속할 수 있도록 한다.

2. 게이트웨이(Gateway)
- 정의:
  - 이종 프로토콜 및 네트워크 간에 통신을 가능하게 하며 다른 네트워크로 들어가는 문 역할을 하는 네트워크 포인트이다.
- 특징:
  - 라우터와는 다르게 다른 프로토콜에서도 서로 통신이 가능하도록 한다.
  - OSI 5계층 이상에서 작동한다.
#### 방화벽(Firewall)과 IDS/IPS
1. 방화벽(Firewall)
- 정의:
  - 미리 정의된 보안 규칙에 기반한, 들어오고 나가는 네트워크 트래픽을 모니터링하고 제어하는 네트워크 보안 시스템이다.
- 특징:
  - 하드웨어나 소프트웨어 형태로 구축할 수 있다.
  - 악의적 트래픽이 네트워크에 도달하기 전에 차단하거나, 중요한 정보가 네트워크를 벗어나지 못하도록 방지한다.

2. IDS/IPS(Intrusion Detection System/Intrusion Prevention System)
   - 정의:
     -  IDS는 네트워크나 시스템에서 발생하는 비정상적인 활동을 감지하고 관리자에게 경고하는 시스템이고, IPS는 이를 확장하여 비정상적인 트래픽을 실시간으로 차단하는 시스템이다.
   - 특징:
     - IDS는 침입 여부를 감지하고 알리는 것, IPS는 침입 이전에 방지하는 것을 목적으로 한다.
     - IDS는 네트워크의 외부, IPS는 네트워크의 내부에 위치한다.
   - 장점:
     - 실시간으로 공격을 탐지, 차단한다.
   - 단점:
     - 잘못된 탐지가 발생할 수 있다.

#### 라우터와 게이트웨이 무선 액세스 포인트(Wireless Access Point)
- 정의:
  - 무선 지원 장치를 유선 네트워크에 연결할 수 있게 해주는 네트워킹 장치이다.
- 특징:
  - 라우터와 달리 네트워크 내부에서 무선 연결 제공에만 중점을 둔다.
  - 라우터 내부에서 사용할 수 있다.

## 네트워크 설계와 관리 (Network Design and Management)

#### 네트워크 설계 원칙 (Principles of Network Design) - 확장성(Scalability), 신뢰성(Reliability), 비용 효율성(Cost Efficiency)
1. 확장성(Scalability)
- 정의:
  - 네트워크는 사용자 수나 트래픽 증가에 따라 쉽게 확장 가능해야 한다.
2. 신뢰성(Reliability)
- 정의:
  - 네트워크는 높은 가용성과 장애 발생 시 빠른 복구 능력을 가져야 한다.
3. 비용 효율성(Cost Efficiency)
- 정의:
  - 네트워크 설계는 산업 표준과 규정을 준수해야 한다.
#### IP 주소 체계 (IP Addressing System)
- 정의:
  - 인터넷에 연결되어 있는 각 장치를 식별하기 위해 TCP/IP 프로토콜 모음의 IP계층에서 사용하는 식별자이다.
- 특징:
  - IP 주소를 이용하여 인터넷 또는 로컬 네트워크를 통해 컴퓨터에 정보를 공유한다.
  - IPv4, IPv6 두 가지의 표준이 있다.
##### IPv4 서브넷팅 및 CIDR(Classless Inter-Domain Routing)
1. 서브넷팅
- 정의:
  - 네트워크를 더 작은 단위의 네트워크로 분할하는 것을 말한다.
- 특징:
  - IP 주소의 일부를 네트워크 주소로 사용하고, 나머지를 호스트 주소로 사용하여 서브넷을 생성한다.
  - 서브넷 마스크를 사용하여 네트워크와 호스트를 구분한다.
- 예시:
  ![image](/image/subnetting.png)
  - 70.12.100.128/26이라는 주소 블럭을 각각 32개, 16개, 16개로 서브네팅한 결과이다.
  - 왼쪽 블럭은 70.12.100.128/27 ~ 70.12.100.159/27 사이의 주소를 사용한다.
  - 가운데 블럭은 70.12.100.160/28 ~ 70.12.100.175/28 사이의 주소를 사용한다.
  - 오른쪽 블럭은 70.12.100.176/28 ~ 70.12.100.191/28 사이의 주소를 사용한다. 

1. CIDR(Classless Inter-Domain Routing)
- 정의:
  - 인터넷상의 데이터 라우팅 효율성을 향상시키는 IP 주소 할당 방법으로, 클래스 없는 주소이다.
- 특징:
  -  IP 주소를 보다 효율적으로 배분하고 IP 공간 낭비를 줄이기 위해 도입되었다.
  -  IP 주소 뒤에 슬래시와 숫자가 붙은 형식으로 나타난다. (예: 192.168.1.0/24)
  -  가변 길이 서브넷 마스킹(VLSM)을 사용하여 IP 주소의 네트워크와 호스트 주소 비트 간의 비율을 변경한다.
- 장점:
  - IP 주소의 낭비를 줄일 수 있다.
  - 서브넷을 통해 데이터를 빠르게 전송할 수 있다.

##### IPv6 주소 체계와 활용
- 정의:
  - 128비트 또는 16바이트 길이의 주소이다.
- 특징
  - 주소 표기는 콜론 16진 표기법으로 16진수의 4자리씩 구분하여 콜론으로 구분하여 표기한다.
  - 유니캐스트 주소, 애니캐스트 주소, 멀티캐스트 주소 총 3가지가 사용된다.
  - 유니캐스트 주소:
    - 일대일 전송 방식에서 사용한다.
    - 일반적인 인터넷 통신, 이메일 전송 등에서 사용한다.
  - 애니캐스트 주소:
    - client와 가장 가까운 PC에서 응답하는 방식에서 사용한다.
    - CDN 서비스에서 사용한다.
  - 멀티캐스트 주소:
    - 특정 그룹의 다수 수신자에게 전송할 때 사용한다.
    - 비디오 스트리밍, 화상 회의 등에서 사용한다.
#### VLAN과 네트워크 분리 (VLAN and Network Segmentation)
- 정의:
  - 가상의 랜으로 하나의 물리 스위치에서 여러 가지 네트워크를 사용하도록 하는 기술이다.
- 특징:
  - 한 스위치에서 서로 다른 논리적 네트워크를 구성할 수 있다.
  - 서로 다른 VLAN에 속한 장치는 통신하지 않기 때문에 보안이 강화된다.
  - 트래픽을 분리하여 혼잡을 줄이고, 네트워크 혼잡을 줄일 수 있다.
  - 스위치에서 설정되고, 스위치가 각 포트에 어떤 VLAN을 할당 할 지 결정한다.
#### NAT(Network Address Translation)와 포트 포워딩 (Port Forwarding)
1.  NAT(Network Address Translation)
- 정의:
  - 하나의 공인 IP 주소를 여러 내부 네트워크 장치가 공유할 수 있도록 해주는 기술이다.
- 특징:
  - 여러 대의 컴퓨터나 장치가 외부 네트워크와 통신할 때 공인 IP 주소 한 개를 사용하고, 내부 네트워크는 사설 IP 주소를 사용하게 된다.
  - 주로 라우터나 방화벽에서 구현된다.
- 사용 목적:
  - 하나의 공인 IP 주소를 여러 장치가 공유하므로, 주소 부족 문제를 해결할 수 있다.
  - 내부 사설 IP는 외부에 공개되지 않고, 외부 네트워크와의 통신에는 동일한 공인 IP를 사용하므로 보안성이 높다.
  - 
2. 포트 포워딩 (Port Forwarding)
- 정의:
  - 컴퓨터 네트워크 상에서 패킷이 방화벽이나 라우터 같은 네트워크 게이트를 지날 때 IP 주소와 포트 번호 결합의 통신 요청을 다른 곳으로 넘겨주는 네트워크 주소 변환이다.
- 특징:
  - NAT 방화벽을 우회하거나, 원격으로 기기에 액세스할 수 있다.
  - 보안성에 취약할 수 있다.
- 사용 목적:
  - 원격으로 다른 PC에 접속
  - IoT 기기를 가정용 네트워크에 연결

#### 트래픽 관리와 QoS (Traffic Management and Quality of Service)
- 정의:
  - 네트워크 트래픽의 우선순위를 관리하여 서비스 품질을 보장하는 기술이다.
- 특징:
  - 실시간 서비스와 비실시간 서비스의 차이를 맞추고 트래픽이 원활히 전달되도록 우선순위를 보장하는데 초점을 맞춘다.

- 네트워크 성능에 영향을 미치는 주요 요소:
  - 대기 시간 (Latency):
    - 패킷이 네트워크를 통해 전달되는 데 걸리는 시간이다.
  - 지연 변동 (Jitter):
    - 패킷이 네트워크를 통해 전달될 때의 지연 시간의 변동이다.
  - 패킷 손실 (Packet Loss):
    - 네트워크에서 패킷이 손실되는 현상이다.
  - 대역폭 (Bandwidth):
    - 네트워크가 데이터를 전송할 수 있는 속도이다.

- QoS 역할
  - IP 패킷의 우선순위 설정
  - 라우터와 스위치의 트래픽 관리

- 예시:
  - VoIP
  - 비디오 스트리밍
  - 기업 네트워크
## 라우팅 및 스위치 (Routing and Switching)

#### 라우팅의 개념 (Routing Concepts)
- 정의:
  - 네트워크에서 데이터를 적절한 경로를 통해 목적지로 전달하는 과정이다.
- 특징:
  - 미리 정해진 규칙을 사용하여 경로를 선택한다.
  - 네트워크가 정체 없이 최대한 많은 용량을 사용할 수 있도록 데이터 트래픽을 관리한다.
  - 
##### 정적 라우팅(Static Routing), 동적 라우팅(Dynamic Routing)
1. 정적 라우팅(Static Routing)
- 정의:
  - 네트워크 관리자가 정적 테이블을 사용하여 네트워크 경로를 수동으로 구성하고 선택하는 방식이다.
- 특징:
  - 네트워크 설계나 파라미터가 일정하게 유지될 것으로 예상되는 경우에 유용하다.
  - 경로가 변경되지 않으므로 안정적이지만, 네트워크 변화에 적응하지 못한다.
- 장점:
  - 간단하고 효율적이다.
  - 네트워크 관리자가 직접 경로를 설정하므로, 외부의 영향을 받지 않아 보안성이 높다.
  - 트래픽 경로를 완전히 통제할 수 있다.
- 단점:
  - 네트워크가 변경될 경우 모든 라우터의 경로를 변경해야 하므로 작업량 증가
  - 대규모 네트워크에서는 비효율적이다.
  - 경로 장애 발생 시 대체 경로를 자동으로 설정할 수 없다.
- 예시:
  - 소규모 사무실의 네트워크 

2. 동적 라우팅(Dynamic Routing)
- 정의:
  - 시스템 내 네트워크의 현재 구성에 맞게 최적의 경로를 찾아서 데이터를 전달하는 방식이다.
- 특징:
  - 라우팅 프로토콜을 사용하여 라우터가 네트워크 상태를 자동으로 학습하고, 최적의 경로를 동적으로 업데이트한다.
  - 네트워크 상태 변화에 따라 실시간으로 경로를 조정한다.
- 장점:
  - 네트워크 변경 사항에 자동으로 적응하므로 관리자가 수동으로 수정할 필요 없다.
  - 대규모 네트워크에서 효과적이다.
  - 새로운 장비와 네트워크 추가가 쉽다.
  - 경로 장애 발생 시 장애 복구가 자동으로 된다.
- 단점:
  - 라우터의 CPU, 메모리, 네트워크 대역폭을 더 소모한다.
  - 초기 설정과 관리가 복잡하다.
- 예시:
  - 대규모 기업 네트워크
#### 라우팅 프로토콜 (Routing Protocols)

##### RIP, OSPF, EIGRP, BGP
1. RIP (Routing Information Protocol)
- 정의:
  - 동적 라우팅의 방식의 프로토콜로, 방향과 기준으로 패킷을 전달하는 프로토콜이다.
- 특징:
  - 거리와 방향으로 길을 찾아가는 Distance Vector 다이나믹 프로토콜이다.
  - Hop count가 적은 경로를 택하여 라우팅하고, Routing Table에 인접 라우터 정보를 저장하여 경로를 결정한다.
  - 최대 Hop count는 15이다.
- 장점:
  - 소규모 네트워크에서 효율적이다.
  - 구성이 간편하다.
- 단점:
  - Hop count가 16 이상이면, 네트워크를 찾지 못해서 데이터를 보내지 못한다.
  - 속도나 거리 지연을 고려하지 않아, 최적의 경로 산정에 비효율적이다.

2. OSPF (Open Shortest Path First)
- 정의:
  - 동적 라우팅의 방식의 프로토콜로, 최단 경로를 기준으로 패킷을 전달하는 프로토콜이다.
- 특징:
  - 최단 경로를 선택하기 위해 다익스트라의 SPF 알고리즘을 사용한다.
  - Area 개념을 사용하여 전체 네트워크를 작은 영역으로 나눈다.
- 장점:
  - Area 개념을 이용하여 효율적인 라우팅 관리가 가능하다.
  - 링크에 상태 변화가 있을 떄, 즉각적으로 Flooding을 하여 반응한다.
- 단점:
  - 초기 구성과 유지 보수가 어렵다.
  - 라우터의 CPU와 메모리 사용량이 높다.

3. EIGRP (Enhanced Interior Gateway Routing Protocol)
- 정의:
  - Cisco사에서 제작한 Cisco 전용 라우팅 프로토콜로, Distance Vector + Link-State이 합쳐진 프로토콜이다.
- 특징:
  - DUAL(Diffusing Update Algorithm) 알고리즘 사용하여 최적 경로와 후속 경로를 선출한다.
  
- 장점:
  - DUAL 알고리즘 사용으로 빠르게 대체 경로를 찾는다.
  - 업데이트가 필요한 라우터에만 전송하여 리소스를 효율적으로 사용한다.
- 단점:
  - Cisco 장비에만 최적화되어, 다른 장비와 호환성이 제한된다.
  - 초기 설정과 유지보수가 어렵다.
- 예시:
  - 대규모 기업 네트워크(Cisco 장비로 구성된 환경)

4. BGP (Border Gateway Protocol)
- 정의:
  - 서로 다른 자율 시스템(AS: Autonomous System)에 속한 라우터들 간에 라우팅 정보를 교환하여 통신을 가능하
- 특징:
  - 관리자가 경로 선택 정책을 정의할 수 있다.
  - AS 경로, 우선순위, 메트릭 등을 기반으로 경로를 선택한다.
- 장점:
  - 인터넷과 같은 글로벌 네트워크에서 필수이다.
  - 경로를 세밀하게 제어할 수 있다.
  - 수백만 개의 경로를 처리할 수 있다.
- 단점:
  - 초기 설정 및 관리가 복잡하다.
  - 경로 변경 시 반영이 느리다.
- 예시:
  - ISP 간의 라우팅
  - 클라우드와 데이터 센터의 연결

#### 스위치와 데이터 전달 (Switching and Data Transmission)

##### MAC 주소 테이블 관리
- 정의:
  - 스위치가 MAC 주소(Media Access Control Address)와 해당 포트(인터페이스) 간의 매핑 정보를 저장하는 데이터베이스 
- 특징:
  - 스위치는 네트워크에서 수신된 프레임의 소스 MAC 주소를 학습하고 이를 해당 포트와 연결하여 테이블에 저장한다.
  - 목적지 MAC 주소를 기반으로 테이블을 조회하여 프레임을 특정 포트로 전송한다.
  - 목적지 MAC 주소가 없거나 브로드캐스트일 경우, 프레임을 모든 포트로 전달한다.
  - 스위치는 네트워크에서의 트래픽을 분석하여 자동으로 MAC 주소를 테이블에 추가한다.
- 장점:
  - 스위치는 MAC 주소 테이블을 기반으로 프레임을 적절한 포트로만 전달하여 트래픽을 최소화한다.
  - 스위치가 자동으로 MAC 주소를 학습한다.
- 단점:
  - 스위치마다 저장할 수 있는 MAC 주소가 제한적이다.
  - 네트워크에 새로운 MAC 주소가 추가되거나, 학습되지 않은 MAC 주소가 있으면 브로드캐스트가 증가한다.
- 예시:
  - 대규모 분산 처리
  - VLAN 환경

##### STP(Spanning Tree Protocol)
- 정의:
  - 스위치 네트워크에서 루프 방지를 위해 사용하는 프로토콜이다.
- 특징:
  - 스위치 네트워크에서 특정 링크를 차단(Block)하여 트리 구조를 형성함으로써 루프 방지한다.
  - 네트워크 장애가 발생하면 차단된 링크를 활성화하여 새로운 경로를 제공한다.
  - IEEE 802.1D 표준으로 정의되었다.
- 장점:
  - 브로드캐스트 폭풍 및 중복 프레임 전달을 방지한다.
  - 네트워크 장애 발생 시 대체 경로를 활성화한다.
- 단점:
  - 네트워크 변경 시 복구 시간이 길어질 수 있다.
  - 루프 방지를 위해 일부 링크를 차단하여 대역폭이 낭비된다.

## 네트워크 보안 (Network Security)

#### 네트워크 위협과 공격 (Network Threats and Attacks)

##### DoS/DDoS, 스니핑(Sniffing), 피싱(Phishing), MITM(Man-In-The-Middle)
1. DoS/DDoS(Denieal of Service/ Distributed Denial of Service)
- 정의:
  - DoS는 서비스 거부 공격으로, 네트워크, 서버, 애플리케이션을 과도한 요청으로 마비시켜 정상적인 서비스를 방해하는 사이버 공격이다.
  - DDoS는 분산 서비스 거부 공격으로, 다수의 장치(봇넷)를 사용하여 대규모로 DoS 공격을 수행하여 목표 시스템의 자원을 고갈시키는 방식이다.
- 특징:
  - DoS 공격은 서버를 공격하여 웹 사이트를 마비시키는 것이 목적이다.
  - DDoS 공격은 여러 대의 컴퓨터나 기계를 이용하여 서버를 공격하는 것이 목적이다.
  - DoS는 단일 위치에서 발생하고, DDoS는 공격의 위치가 여러 곳이여서 출처를 찾기 어렵다.
  - DDoS는 여러 위치에서 시작하므로, 전파 속도가 매우 빠르다.
- 공격 종류:
  - TearDrop: 수많은 인터넷 프로토콜(IP) 데이터 조각을 네트워크로 전송하는 DoS 공격이다.
  - Flooding: 서버에 여러 개의 연결 요청을 전송한 다음 핸드셰이크를 완료하기 위한 응답을 하지 않는 DoS 공격이다.
  - IP Fragmentation: 수신 네트워크가 재조합할 수 없는 변경된 네트워크 패킷을 제공하는 일종의 DoS 공격이다.
  - Volumetric Attack: 대역폭 리소스를 표적으로 삼는 데 사용되는 DDoS 공격의 한 유형이다.

2. 스니핑(Sniffing)
- 정의:
  - 네트워크 상에서 데이터 패킷을 가로채거나 분석하는 행위이다.
- 특징:
  - 주어진 네트워크를 통과하는 모든 데이터 패킷을 모니터링하고 캡처하는 프로세스이다.
- 공격 종류:
  - 스위치 공격: 네트워크 스위치가 MAC 주소 테이블을 기반으로 포트에 패킷을 스위칭할 때 정상적인 스위칭 기능을 마비시키는 공격이다.
  - ARP 우회 공격: 위조된 ARP의 응답 패킷을 네트워크에 주기적으로 브로드캐스트 하면서 네트워크 상의 다른 모든 호스트들이 공격자 호스트를 라우터로 간주하여 네트워크 상의 모든 트래픽이 공격자의 호스트를 경유하도록 함으로써 훔쳐보는 공격이다.
  - ICMP 우회 공격: 라우터가 패킷이 잘못 라우팅 되고 있다고 판단하고 동일한 대상으로 전송되는 후속 패킷에 대해 다른 라우터를 사용해야 한다고 발신자에게 알리고자 할 때 사용되는 공격이다.

3. 피싱(Phishing)
- 정의:
  - 피싱은 사기성 이메일, 문자 메시지, 전화 또는 웹사이트를 사용하여 사람들을 속여 민감한 데이터를 공유하거나 맬웨어를 다운로드하거나 기타 사이버 범죄에 노출되도록 하는 사이버 공격이다.
- 특징:
  - 인간 심리를 악용하여 사용자가 의심하지 않고 행동하도록 유도한다.
  - 이메일, 웹사이트, 메시지를 신뢰할 수 있는 기관, 기업, 혹은 개인처럼 위조한다.
  - 계정 로그인 정보, 신용카드 정보, 금융 계좌 비밀번호 등을 탈취한다.
- 공격 종류:
  - 이메일 피싱 (Email Phishing): 대량으로 발송되는 피싱 이메일로 정보를 탈취한다.
  - 스피어 피싱 (Spear Phishing): 특정 개인이나 조직을 타겟으로 정교하게 설계된 피싱 공격이다.
  - 스미싱 (Smishing): 문자 메시지(SMS)를 통해 링크를 클릭하거나 정보를 제공하도록 유도한다.

4. MITM(Man-In-The-Middle)
- 정의:
  - 공격은 공격자가 두 당사자 간의 통신을 가로채거나 조작하여 민감한 정보를 탈취하거나 악의적인 행위를 수행하는 사이버 공격이다.
- 특징:
  - 공격자는 사용자와 서버(또는 두 사용자의 디바이스) 사이에 위치하여 통신을 가로챈다.
  - 데이터 도청, 암호 탈취, 금융 정보 수집 등이 가능하며, 데이터를 변조하거나 악성 코드를 삽입할 수 있다.
  - 보안이 취약한 네트워크(예: 공개 Wi-Fi)에서 발생하기 쉽다.
- 공격 종류:
  - Wi-Fi 스니핑 (Wi-Fi Eavesdropping): 암호화되지 않은 HTTP 트래픽이 타겟이다.
  - IP 스푸핑 (IP Spoofing): 신뢰할 수 있는 IP로 가장하여 통신 세션에 침입한다.
  - ARP 스푸핑 (ARP Spoofing): 로컬 네트워크에서 ARP(주소 결정 프로토콜) 캐시를 조작하여 공격자의 MAC 주소를 타겟의 IP 주소와 연결한다.

#### 보안 기술 (Security Technologies)

##### 암호화 - 대칭키와 비대칭키
1. 대칭키(Symmetric Key)
- 정의:
  - 암호화와 복호화의 키가 동일한 암호화 방식이다.
- 특징:
  - 암호문의 수신자와 송신자가 같은 키를 사용하므로, 키를 안전하게 공유하는 것이 중요하다.
  - 계산이 단순하여 속도가 빠르다.
- 장점:
  - 암복호화 처리 속도가 빠르다.
  - 대량 데이터를 처리할 때 유리하다.
- 단점:
  - 키를 안전하게 공유하는 것에 어려움이 있다.
  - 키가 노출되면 데이터 값도 위험에 처한다.
- 예시:
  - AES(Advanced Encryption Standard)
  - DES(Data Encryption Standard)
  - 3DES(Triple DES)

2. 비대칭키(Asymmetric Key)
- 정의:
  - 암호화와 복호화에 두 개의 다른 키를 사용한다.
- 특징:
  - 공개키(Public Key)를 통해 데이터를 암호화한다.
  - 개인키(Private Key)를 통해 데이터를 복호화한다.
  - 대칭 키 방식의 키 공유 문제를 해결한다.
- 장점:
  - 키 공유가 안전하고, 공개키는 노출되어도 무관하다.
- 단점:
  - 계산이 복잡하고 느리다.
  - 대량 데이터 암호화에 비효율적이다.
- 예시:
  - RSA(Rivest-Shamir-Adleman)

##### SSL/TLS, IPSec
1. SSL/TLS
- 정의:
  - SSL(Secure Sockets Layer)란 클라이언트와 서버 간의 데이터 전송을 암호화하여 도청 및 위조를 방지하는 기술이다.
  - TLS (Transport Layer Security)란 SSL의 개선판으로, 현재 웹 보안의 표준 프로토콜이다.
- 특징:
  - TLS가 사용되기 전에는 SSL이 대신 역할을 하였다.
  - Hand-shake 프로세스를 용이하게 하고 브라우저와 웹 서버 간에 암호화된 통신을 설정하는 인증서를 사용한다.
  - RSA 키 교환 알고리즘을 사용한다.
  - 전송 계층에 위치한다.
- 예시
  - HTTPS, 이메일 , VoIP
2. IPsec (Internet Protocol Security)
- 정의:
  - 네트워크에서의 안전한 연결을 설정하기 위한 통신 규칙 또는 프로토콜 세트이다.
- 특징:
  - 네트워크 계층에 존재한다.
  - IP패킷을 암호화하고 인증하는 보안을 위한 표준이다.
  - VPN을 구현하는 데 사용된다.
- 예시:
  - VPN
#### 방화벽과 ACL(Access Control List)
- 정의:
  - 네트워크 트래픽이나 리소스에 대한 접근을 제어하는 허용, 차단에 대한 규칙 목록이다.
- 특징:
  - 허가(Permit)되지 않은 이용자가 라우터나 네트워크에 접근하려고 하는 것을 차단한다.
  - 송,수신지 주소, 포트 번호, 프로토콜 등으로 특정 패킷을 필터링 할 수 있다.
  - 특정 사용자, 서비스를 제한하거나 라우팅 경로의 조정이 필요한 경우에 사용한다.
- 예시:
  - 외부 IP 차단 및 내부망 트래픽 제어
  - 기업 직원의 인터넷 사용 제어
#### VPN (Virtual Private Network)
- 정의:
  - 공용 네트워크(예: 인터넷)를 통해 안전하게 데이터를 전송하고, 원격 네트워크 또는 장치가 마치 같은 사설 네트워크에 있는 것처럼 연결하도록 하는 기술이다.
- 특징:
  - 데이터를 암호화하여 전송하고, 도청을 방지한다.
  - 데이터 패킷을 보호하며, 중간자 공격(MITM)을 방지한다.
- 예시:
  - 사내 네트워크에 접근
  - 특정 국가에서 차단된 컨텐츠 접속
#### Zero Trust 네트워크 (Zero Trust Network)
- 정의:
  - 네트워크 내부와 외부의 모든 요청에 대해 "절대 신뢰하지 않고 항상 검증"하는 원칙을 따르는 네트워크 보안 모델이다.
- 특징:
  - 네트워크 경계 내에 있다고 해서 신뢰를 부여하지 않는다.
  - 분산된 네트워크 환경에서도 안전한 접근을 보장한다.
- 예시:
  - 클라우드 인프라 보호
  - 원격 근무 환경 설정
## 네트워크 성능 평가 및 최적화 (Network Performance Evaluation and Optimization)

#### 네트워크 성능 지표 (Network Performance Metrics)

##### 대역폭(Bandwidth), 지연(Latency), 처리량(Throughput)
1. 대역폭(Bandwidth)
- 정의:
  - 네트워크 연결을 통해 전달할 수 있는 데이터의 양
- 특징:
  - 주로 초당 비트수(bps) 단위로 측정한다.
  - 대역폭이 크면 전송 속도가 빨라진다.

2. 지연(Latency)
- 정의:
  - 데이터가 송신자에서 수신자까지 도달하는 데 걸리는 시간이다.
- 특징:
  - 네트워크의 응답 시간을 측정하는 지표이다.

3. 처리량(Throughput)
- 정의:
  - 네트워크가 실제로 처리할 수 있는 데이터의 양이다.
- 특징:
  - 네트워크를 통해 얼마나 많은 양의 데이터가 전송될 수 있는 지 나타낸다.
#### 네트워크 트래픽 분석 (Traffic Analysis)
- 정의:
  -  네트워크를 통해 흐르는 데이터를 모니터링하고, 이를 분석하여 네트워크 성능, 보안, 문제 해결, 최적화 등을 위한 유용한 정보를 도출하는 과정이다.
- 특징:
  - 성능을 모니터링하고, 최적화 하는 것이 목적이다.
  - 주로 흐름 데이터, 패킷 데이터를 가지고 정보를 수집한다.
  
#### 부하 분산과 로드 밸런싱 (Load Balancing)
1. 부하 분산
- 정의:
  - 두 대 이상의 컴퓨터 간에 계산 작업 부하를 분산하는 작업이다.
- 특징:
  - 네트워크 트래픽을 여러 서버로 분산시켜, 서버의 부담이 줄어들고 성능이 빨라진다.
  - 서버 및 네트워크의 과부하를 피하고, 시스템의 효율을 높이기 위한 목적으로 사용된다.
  - 특정 서버에 문제가 발생해도, 전체 서비스에 미치는 영향을 최소화할 수 있다.

2. 로드 밸런싱
- 정의:
  - 애플리케이션을 지원하는 리소스 풀 전체에 네트워크 트래픽을 균등하게 배포하는 방법이다.
- 특징:
  - 애플리케이션 서버와 방문자 또는 클라이언트 간의 인터넷 트래픽을 지시하고 제어한다.
  - 오토 스케일링을 하여 조건에 맞추어 서버 용량을 변경한다.
## 심화 주제
#### SDN(Software-Defined Networking)과 NFV(Network Function Virtualization)
1. SDN(Software-Defined Netwrorking)
- 정의:
  - 네트워크 관리와 제어를 중앙 집중식으로 소프트웨어를 통해 설정, 모니터링, 관리하는 네트워크 아키텍처이다.
- 특징:
  - 소프트웨어 애플리케이션과 API를 이용하여 네트워크를 프로그래밍하고, 중앙에서 전체 네트워크를 관리한다.
  - 네트워크 장비의 제어부와 전송부를 분리한다.
  - 장비 관리에 대한 비용을 절감할 수 있고, 네트워크를 유연하게 확장할 수 있다.

2. NFV(Network Function Virtualization)
- 정의:
  - 네트워크 기능을 추상화하여 표준화된 컴퓨팅 노드에서 실행되는 소프트웨어를 통해 네트워크 기능을 설치, 제어 및 조작하도록 지원한다.
- 특징:
  - 네트워크 기능을 VNFs라는 독립적인 소프트웨어 애플리케이션으로 변환하여 실행한다.
  - VNF, NFVI, NFV-MANO 라는 프레임워크가 존재한다.

#### MPLS(Multi-Protocol Label Switching) 기술
- 정의:
  - 네트워크 트래픽을 효율적으로 관리하고, 고속으로 전송하기 위해 레이블(Label)을 사용하는 기술이다.
- 특징:
  - 레이블을 통해 패킷을 처리하여, 라우팅 방식에 비해 성능이 크게 향상된다.
  - 트래픽 흐름을 최적화하는 기능을 제공한다.
  - QoS 기능을 제공한다.
  - MPLS VPN을 구현할 수 있다.
